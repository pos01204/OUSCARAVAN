### 도움말 페이지 진단 & 고급 호텔 컨셉 개선 제안서

- **대상**: `app/guest/[token]/help` (고객 도움말)
- **컨셉**: **고급 호텔(Concierge) 톤** — 차분한 여백, 명확한 정보 위계, “필요할 때만 강해지는” 긴급성
- **목표**: 복잡도를 낮추면서도(인지부하↓) 행동까지 빠르게(탭 수↓) 연결
- **작성 기준(자체 진단)**:
  - **정보구조/탐색성**: Progressive Disclosure, Scanability, Hick’s Law
  - **시각적 위계**: 대비(contrast)·크기·간격·밀도·그룹화(Gestalt)
  - **접근성**: WCAG(텍스트 대비/터치타깃/키보드), Screen Reader semantics
  - **상호작용**: 오작동 방지(특히 전화), 상태 피드백, motion hygiene
  - **성능/안정성**: Hydration 비용, 렌더링, 동적 데이터/오프라인
  - **운영/분석**: 이벤트 로깅, A/B, 원격 설정(운영팀 컨트롤)

---

### 1) 현재 화면 진단(문제 원인)

#### **A. 구분(Sectioning)이 약해 “한 덩어리”로 보임**
- **증상**: `Emergency / Nearby / FAQ / Contact`가 라벨만 바뀌고 **같은 바탕/같은 레벨의 카드**로 연속 배치되어, 스크롤 시 “리스트”처럼 느껴짐.
- **결과**: 사용자는 “지금 내가 어디를 보고 있는지”를 계속 다시 파악해야 함(인지부하↑).

#### **B. 위계(Hierarchy)가 단조로워 중요도 차이가 안 보임**
- **응급(119/112)**는 시각적으로 강하지만(다크 카드), **그 아래 섹션들과의 대비 설계가 없음** → 응급이 끝난 뒤도 여전히 비슷한 밀도/리듬으로 이어짐.
- **FAQ**는 “중요 질문”이 섞여 있어도 **동일한 리스트 룩** → 가장 중요한 정보(체크인 등)의 발견성이 낮음.

#### **C. CTA(행동) 명확성 부족**
- `119/112`는 “전화”가 암시되지만, **주변시설/FAQ/관리자 연락의 행동 목적이 서로 다른데 동일한 컴포넌트 계열**로 보임.
- “전화/지도/상세보기/펼치기”가 섞여 있어 **한 화면에서 행동 규칙이 통일되지 않음**.

#### **D. 전화 UX는 사고(오탭) 리스크가 큰 영역**
- FAB/응급 카드 모두 **즉시 `tel:` 실행**은 오탭 비용이 큼(특히 이동 중/한 손 조작).
- “긴급(119/112)”은 즉시 연결이 타당하지만, “관리자”는 **확인/지연/대체 채널 안내**가 더 적절할 수 있음.

---

### 2) 고급 호텔 컨셉을 위한 디자인 시스템(페이지 전용)

#### **핵심 원칙**
- **Quiet Luxury**: 기본은 차분(Neutral), 강조는 “필요한 순간에만” (Emergency).
- **Layering**: 배경(페이지) → 섹션 컨테이너 → 카드 → CTA의 계층을 분명히.
- **Density Control**: 한 화면에 보여주는 “서로 다른 행동”의 수를 제한.

#### **권장 토큰(테일윈드 기반)**
- **Page background**: `bg-neutral-50`
- **Section surface**: `bg-white/70 backdrop-blur` (또는 `bg-white` + subtle border)
- **Border**: `border-neutral-200/70`
- **Shadow**: `shadow-[0_8px_24px_rgba(0,0,0,0.06)]` (과한 그림자 금지)
- **Text**: title `text-neutral-900`, body `text-neutral-600`, meta `text-neutral-400`
- **Emergency accent**: 빨강/파랑 원색 금지 → **초저채도 틴트**(예: `bg-red-50/40`, `bg-blue-50/40`) + 아이콘/텍스트만 제한적으로.

---

### 3) 컴포넌트/레이아웃 개선안(“구분”을 강하게 만들기)

#### **A. 페이지를 ‘섹션 컨테이너’로 감싸기(가장 큰 효과/낮은 위험)**
- **문제**: 현재는 섹션 라벨 + 콘텐츠가 이어지는 구조라 “블록”이 약함
- **개선**: 각 섹션을 `HelpSection`으로 감싸 **배경/패딩/분리선/헤더/보조설명**을 표준화

**제안 컴포넌트**
- `HelpPageShell`: 페이지 배경/최대폭/하단 안전 여백(FAB 영역) 통일
- `HelpSection`: 섹션 헤더(타이틀/설명/우측 액션) + 섹션 surface
- `HelpCard`: 공통 카드 스타일(모서리/보더/그림자/hover)
- `HelpActionRow`: “행동 규칙”을 표준화한 row(아이콘/타이틀/서브/우측 행동)

#### **B. ‘행동 유형’별로 UI를 분리(전화 vs 지도 vs 펼치기)**
- **전화형(긴급)**: `EmergencyActionCard` (큰 타깃, 1탭 연결, 단 오탭 방지 옵션)
- **지도형(주변시설)**: `LocationTile` (주소/거리/“지도 열기” CTA)
- **지식형(FAQ)**: `FAQAccordion` (펼치기/접기 + “관련 링크”)
- **연락형(관리자)**: `ConciergeContactCard` (전화 + 운영시간/응답 기대치/대체 채널)

#### **C. 정보 구조(IA) 재배치**
- **상단(즉시 행동)**: Emergency (119/112)만 최상단 고정
- **중단(빠른 해결)**: FAQ(Top 3) + “더 보기” 링크
- **하단(보조/대체)**: Nearby, Contact
- **옵션**: “긴급이 아니라면” 안내 문구로 톤 조절(고급 호텔 컨셉)

---

### 4) 전화 UX(효과/오작동 방지/신뢰감) 개선안

#### **A. ‘관리자 전화’는 실수 방지 UX 적용(난이도: 중~상)**
- **옵션 1: Hold-to-call(1초 길게 누르기)**  
  - 장점: 오탭 방지, 고급스러운 상호작용  
  - 구현: Pointer events + progress ring(1초) + 완료 시 `tel:` 실행
- **옵션 2: 확인 바텀시트(Confirm Sheet)**  
  - 장점: 구현 쉬움, 정보 제공 가능(“운영시간/긴급시 119 안내”)  
  - 구현: `Dialog/Sheet` + primary CTA “전화 걸기”

> 권장: **관리자=Confirm Sheet**, **119/112=즉시 연결** (행동 규칙 분리)

#### **B. ‘전화 전 상태 피드백’**
- 버튼 터치 시:
  - **미세 햅틱 대체**: `navigator.vibrate(10)` (지원 환경에서만, 실패해도 무해)
  - **상태 토스트**: “전화 앱으로 연결 중…” (1~2초)
  - **접근성**: `aria-live="polite"`로 상태 안내

#### **C. 로그/분석(운영 난이도: 상)**
- 이벤트: `help_call_intent`, `help_call_confirm`, `help_call_cancel`, `help_call_outbound`
- “오탭률”을 지표로 관리 (confirm open 대비 confirm/cancel 비율)

---

### 5) 동적 효과(고급스럽게, 과하지 않게) 개선안

#### **A. 스크롤 기반 섹션 인식(난이도: 상)**
- **Sticky Section Header + Scrollspy**
  - 현재 보고 있는 섹션을 상단에 작은 라벨로 고정
  - 우측에 “맨 위/FAQ로/긴급으로” 점프 버튼(상황별)
- 구현: `IntersectionObserver` + 상태 저장

#### **B. ‘상태의 모션’만 사용(장식 모션 최소화)**
- 원칙: “모션은 정보 전달/행동 유도에만”
- 적용:
  - 카드 hover/press: 150ms, `ease-out`
  - 아코디언: height 애니메이션 + opacity (prefers-reduced-motion 대응)
  - 섹션 진입: 1회 fade-up(옵션) — 과하면 제거

#### **C. ‘사용자 진행도’ 저장(난이도: 상)**
- FAQ에서 펼쳤던 항목/최근 본 섹션을 **로컬 저장**  
  - 재방문 시 “이전에 보던 질문 이어보기” 제공
  - 구현: `localStorage` + 버전 키(구조 변경 대응)

---

### 6) “난이도 높은” 확장 과제(운영/개발 가치 큰 것 위주)

#### **A. 원격 설정(Remote Config) 기반 도움말 운영(난이도: 상~최상)**
- 목적: 운영팀이 “오늘 자주 묻는 질문”/“긴급 공지”를 배포할 수 있게
- 구현 방향:
  - JSON 스키마(FAQ/Notice/Emergency policy) 정의
  - 캐시 전략(ETag/TTL) + 실패 시 로컬 fallback
  - 관리자/운영자 UI가 있다면 연동(추후)

#### **B. 개인화(예약 토큰 기반) 컨텍스트 도움말(난이도: 상)**
- 예: 체크인 당일이면 FAQ 상단을 “체크인/입실” 중심으로 재정렬
- 입력: 예약 날짜/상태(서버에서 이미 검증됨)
- 출력: 섹션 순서/FAQ priority 동적 계산

#### **C. 오프라인/저신뢰 네트워크 대비(난이도: 상)**
- 응급 연락처는 **항상 접근 가능**해야 함
- 구현:
  - Service Worker(선택) 또는 `localStorage`에 핵심 연락처 프리캐시
  - 지도 링크는 오프라인일 때 “주소 텍스트/복사” 제공

#### **D. A/B 테스트 프레임워크(난이도: 최상)**
- 실험 예:
  - 관리자 연락: **FAB vs 하단 카드**
  - FAQ: **Top3 + 더보기 vs 전체 노출**
  - 관리자 전화: **confirm vs hold-to-call**
- 필요:
  - 실험 할당(토큰 기반 고정), 이벤트 수집, 지표 대시보드(최소 로그)

#### **E. 접근성 자동화(난이도: 중~상)**
- `axe` 기반 검사 + CI에 포함(가능하면)
- 키보드 포커스 링/탭 순서/스크린리더 레이블 정비

---

### 7) 구현 로드맵(추천 순서)

#### **Phase 1 (효과 큼 / 위험 낮음)**
- **HelpPageShell + HelpSection 도입**으로 섹션 경계/여백/표준화
- “행동 규칙” 정리: 전화/지도/펼치기 컴포넌트 분리

#### **Phase 2 (UX 품질 상승 / 난이도 중)**
- 관리자 전화 **Confirm Sheet** + 이벤트 로깅
- FAQ “Top3 + 더 보기” + 펼침 상태 저장

#### **Phase 3 (난이도 상)**
- Scrollspy + Sticky section header
- 개인화(priority 동적 재정렬)

#### **Phase 4 (난이도 최상 / 운영 가치)**
- Remote Config + 캐시/롤백 + A/B 테스트
- 오프라인 대비(핵심 연락처 프리캐시)

---

### 8) 성공 지표(측정 가능하게)

- **탐색성**
  - 도움말 진입 후 \(5초 이내\) 응급/FAQ 클릭률
- **오탭 감소**
  - 관리자 전화 confirm 열림 대비 cancel 비율, 즉시전화 실행률
- **자기해결**
  - FAQ 펼침 후 “관리자 전화” 클릭률 감소(정책에 따라 목표 조정)
- **성능**
  - LCP/CLS, 상호작용 지연(특히 아코디언)

---

### 9) 결론

현재 복잡해 보이는 핵심 이유는 **섹션 컨테이너(구분) 부재 + 행동 유형 혼재**입니다.  
고급 호텔 컨셉에서는 “강한 색”이 아니라 **레이어/여백/규칙**으로 위계를 만들고, 전화는 “즉시 실행”이 아니라 **상황별 안전장치 + 신뢰감 있는 피드백**이 핵심입니다.
