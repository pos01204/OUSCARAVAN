## ♿ 접근성 (A11y) 현황

### 현재 구현된 접근성 기능

#### 1. 시맨틱 HTML
- ✅ `<nav>`, `<main>`, `<header>`, `<footer>` 태그 사용
- ✅ 적절한 HTML 구조

#### 2. 키보드 네비게이션
- ✅ 모든 버튼과 링크가 키보드로 접근 가능
- ✅ 폼 요소가 키보드로 접근 가능
- ⚠️ 포커스 인디케이터 개선 필요 (일부 요소)

#### 3. ARIA 레이블
- ⚠️ 일부 버튼과 링크에 ARIA 레이블 부족
- ⚠️ 아이콘 버튼에 aria-label 필요

#### 4. 색상 대비
- ✅ Tailwind CSS 기본 색상 사용 (대비 고려)
- ⚠️ WCAG AA 기준 확인 필요

#### 5. 스크린 리더 지원
- ✅ 시맨틱 HTML 사용
- ⚠️ ARIA 레이블 추가 필요

### 개선 필요 사항

#### 1. 관리자 레이아웃
- [ ] 네비게이션 메뉴에 ARIA 레이블 추가
- [ ] 로그아웃 버튼에 aria-label 추가
- [ ] 모바일 메뉴 토글 버튼 추가 (햄버거 메뉴)

#### 2. 관리자 페이지 전반
- [ ] 아이콘 버튼에 aria-label 추가
- [ ] 포커스 인디케이터 개선
- [ ] 키보드 단축키 안내 (선택사항)

#### 3. 고객 페이지 전반
- [ ] 하단 네비게이션에 ARIA 레이블 추가
- [ ] 카드 요소에 적절한 role 속성 추가
- [ ] 이미지에 alt 텍스트 확인
