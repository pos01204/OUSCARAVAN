# OUSCARAVAN 카드 비주얼 리팩토링 기획안 (촌스러움 제거 · 완성도 극대화)

> **작성일**: 2026-01-24  
> **목표**: 현재 프레임/레이아웃/기능 흐름은 유지하면서, 카드 컴포넌트의 **색상·표면(서피스)·그림자·동적 요소·전환 규칙**을 재정의해 “프리미엄/모던” 인상을 만든다.  
> **범위**: 고객/관리자 공통 카드 시스템 + 고객 전용 마이크로 인터랙션/전환(관리자 최소)  
> **전제**: **다크모드 미지원**(라이트 고정), 기존 브랜드 톤(크림/따뜻한 다크) 유지

---

## 1) 현재 카드가 “촌스러워 보이는” 핵심 원인(진단)

### 1.1 색이 ‘브랜드색 = 배경색’처럼 과하게 퍼짐
- 크림/베이지가 카드 배경/테두리/악센트에 동시에 넓게 쓰이면 **낡은 UI(올드 톤)**로 체감됩니다.
- 해결: 브랜드색은 **넓은 면이 아니라 “엣지/포인트”**로 사용하고, 면은 **중립(뉴트럴)**로 정리합니다.

### 1.2 표면 계층(surface) 구분이 약해서 “종이 뭉치”처럼 보임
- 카드가 다 비슷한 흰/베이지면 + 비슷한 border면, **중요도/깊이**가 없고 촌스럽게 보입니다.
- 해결: Surface를 3단(페이지/카드/인셋)로 분리하고 **인셋(inside)**에는 톤을 낮춘 배경을 사용합니다.

### 1.3 그림자/테두리가 “둘 다 강하게” 걸려 미묘하게 둔탁함
- border가 강하고 shadow도 강하면 **구형 머티리얼**처럼 보입니다.
- 해결: 기본은 **얇은 헤어라인 + 아주 부드러운 그림자**, 강조/호버에서만 단계 상승.

### 1.4 동적 요소가 “규칙” 없이 여기저기 달라서 마감이 약함
- hover/active/전환이 컴포넌트별로 다르면 “임시로 붙인 효과” 느낌이 납니다.
- 해결: **Motion 토큰(시간/이징/거리)**과 **카드 인터랙션 규칙**을 고정합니다.

---

## 2) 디자인 방향(한 문장)

**Warm Modern Minimal**: 면은 뉴트럴로 정돈하고, 브랜드 크림은 “엣지/포인트/하이라이트”로만 사용한다. 고객은 부드러운 모션으로 생동감을, 관리자는 스캔성을 해치지 않는 최소 인터랙션만 제공한다.

---

## 3) 색상/표면(서피스) 시스템 재정의 (토큰 기반)

현재 `tailwind.config.ts`에는 이미 라이트 토큰이 잘 잡혀 있습니다.
여기서 “촌스러움”을 줄이기 위해 **서피스 계층 + 브랜드 사용 규칙**을 추가합니다.

### 3.1 서피스 계층(권장)
- **Surface / Page**: `background` (페이지 바탕)
- **Surface / Card**: `background-elevated` (카드 면)
- **Surface / Inset**: `background-muted` 또는 `muted.DEFAULT` (카드 내부 박스)
- **Surface / Accent wash(선택)**: *가급적 배제*하고, 필요 시에도 “한 군데(고객 Hero/대표 CTA)”에만 제한 적용

### 3.2 브랜드 색 사용 규칙(촌스러움 방지)
- **대면적 배경**: 뉴트럴만 사용(화이트/연회색)
- **브랜드 크림/베이지**:  
  - 카드 테두리 “엣지” (`border-brand-cream-dark/25~45`)  
  - 상단 2px 스트로크(“헤어라인”) 또는 아이콘 타일 배경처럼 **좁은 영역**  
  - CTA 카드의 “아주 약한 wash”
- **경고/에러**: 빨강 배경을 크게 깔지 말고, **좌측 스트립 + 텍스트/아이콘** 중심(관리자 포함)

### 3.3 대비(가독성) 기준
- 본문 텍스트: `brand-dark` / `foreground` 유지
- 보조 텍스트: `muted-foreground` 사용, `opacity`로 흐리기보다는 **토큰으로 통일**
- 아이콘 컬러: 채도 높은 색을 줄이고 `status-*`를 “점/라인/아이콘” 정도로 제한

### 3.11 그라데이션 정책(중요) — ‘세련됨’ 유지용 가드레일
> 그라데이션은 잘못 쓰면 가장 빠르게 “촌스러움”을 유발합니다.  
> 이 프로젝트의 기본 원칙은 **그라데이션을 기본 도구로 쓰지 않는다** 입니다.

#### A) 기본 원칙(Do/Don't)
- **Do**
  - 그라데이션은 **“제품 서명 영역(Signature)”**에서만 사용(고객 홈 Hero, 대표 CTA 1곳)
  - 컬러는 **뉴트럴 ↔ 뉴트럴** 또는 **뉴트럴 ↔ 아주 옅은 크림**만
  - 대비는 **미세(명도 차 2~6% 수준)**, 방향도 **단일 방향**만
  - “배경 장식”은 1~2개 요소로 제한(블러/노이즈 포함)
- **Don’t**
  - 카드마다 서로 다른 그라데이션(통일성 파괴)
  - 다색(3색 이상), 고채도 코랄/오렌지 면적 그라데이션
  - 라디얼 그라데이션 남발(레트로/유치해 보일 확률 높음)
  - 테두리+진한 그림자+그라데이션을 동시에 사용(구형 머티리얼 느낌)

#### B) 권장 대체 수단(그라데이션 없이도 고급스러움 만드는 법)
- **헤어라인 스트로크**: 카드 상단 1px/2px 라인(아주 옅은 `brand-cream-dark/25~35`)
- **서피스 대비**: Page(연회색) ↔ Card(화이트) ↔ Inset(톤다운)으로 계층을 만든다
- **소프트 섀도우**: 기본 약하게, hover에서만 한 단계 상승
- **아이콘 타일**: 뉴트럴 타일 + 아이콘 컬러만(채도 면적 최소화)
- **마이크로 인터랙션**: “짧은” lift/press/fade로 프리미엄 감(색으로 때우지 않음)

#### C) “허용 목록(Whitelist)” 예시(필요할 때만)
- 고객 홈 Hero(단 1개):
  - `bg-gradient-to-br from-background-elevated to-background`
  - 또는 `from-background-elevated via-background-elevated to-background` 같은 **거의 안 느껴지는** 수준
- 대표 CTA(단 1개, 필요 시):
  - `bg-gradient-to-br from-background-elevated to-muted/40`

> **주의**: 위 예시 외 그라데이션을 추가하려면 “통일성 체크리스트(11장)”에 먼저 통과해야 합니다.

### 3.4 배경(Page) vs 컴포넌트(Card) 구분 규칙 (Separation)
> “촌스러움”은 보통 **바탕과 카드가 같은 톤**이거나, **구분선/배경색을 남발**할 때 발생합니다.  
> 아래 규칙은 “페이지는 조용하게, 컴포넌트가 주인공”이 되도록 강제합니다.

- **Page(바탕) 원칙**
  - 기본 바탕은 항상 `background` (무채색/연회색)로 고정
  - 페이지 바탕에는 **테두리/카드 같은 박스감 금지**(= 페이지 전체가 카드처럼 보이면 구형 느낌)
  - 배경 장식(그라데이션/블러)은 고객 홈의 Hero처럼 **1~2개 포인트 섹션**에만 제한 적용

- **Card(면) 원칙**
  - 카드 면은 항상 `background-elevated`(화이트) 기반
  - 카드 구분은 “색면”이 아니라 **헤어라인 border + 소프트 shadow**로 만든다(기본은 약하게)
  - 카드 내부 정보 박스는 `background-muted`(인셋)로 통일해 “카드 안의 카드”를 방지

- **구분을 만드는 도구 우선순위**
  - 1순위: **여백(Spacing)** / 정렬 / 타이포 위계
  - 2순위: 헤어라인 border(낮은 불투명도)
  - 3순위: 소프트 shadow(hover에서만 한 단계 상승)
  - 4순위: 배경색/컬러 wash(고객 CTA에만 아주 약하게)
  - 금지: 굵은 테두리, 진한 그림자, 넓은 베이지 배경면(올드 톤 유발)

- **레이어(깊이) 최대 규칙**
  - 한 화면에서 보이는 “깊이 단계”는 최대 3단까지만 허용
    - Page → Card → Inset
  - Inset 안에 또 다른 Card(= Card-in-Card)는 원칙적으로 금지(필요하면 borderless block)

### 3.5 UI 규칙(Spacing) — 8pt 그리드로 강제 통일
- **기본 단위**: 8px(필요 시 4px 보조), 임의의 10/12/14px 남발 금지
- **섹션 간격(페이지)**: `space-y-6`를 기본(모바일), 데스크톱은 `md:space-y-8`까지
- **카드 패딩(권장)**
  - Card: `p-4` 기본, 밀도 낮추는 섹션은 `p-5~6`
  - Header/Body 간격: `pb-3`(Header), Body는 `space-y-3~4`
  - Inset 박스: `p-3~4`, radius는 카드보다 한 단계 작게(`rounded-xl`)
- **리스트 카드**: 항목 간격은 12px처럼 “애매한 간격” 대신 `space-y-3`로 고정

### 3.6 UI 규칙(Typography) — “스캔 가능한 위계”
- **타이틀(H1/H2)**: 굵게/크게, 1줄 우선(넘치면 줄바꿈보다 “짧은 2줄”까지만)
- **카드 타이틀**: `text-base~lg + font-semibold` 선에서 통일(굵기 과다 금지)
- **메타/설명**: 항상 `text-xs~sm + text-muted-foreground`
- **숫자(가격/KPI)**: `font-black`은 “핵심 1개 영역”에만(여러 곳에 남발 금지)
- **행간**: 안내 텍스트는 `leading-relaxed`, 목록/라벨은 기본 행간

### 3.7 UI 규칙(Iconography) — “아이콘 타일” 통일
- **아이콘 크기**: 16/20/24px만 사용(`h-4`, `h-5`, `h-6`)
- **스트로크**: `strokeWidth`는 동일 규칙(예: 2 또는 2.5)로 통일
- **아이콘 타일(배경)**
  - 배경은 **뉴트럴(연회색/연화이트)**, 컬러는 아이콘에만
  - 채도 높은 타일(파랑/초록/보라 면적)을 남발하지 않는다(유치해 보이기 쉬움)

### 3.8 UI 규칙(State & Feedback) — 상태 표현 단일 규칙
- **상태는 컴포넌트로만 표현**
  - 상태 텍스트: `StatusPill` / `StatusBadge`
  - 빈 상태: `EmptyState`
  - 에러: `ErrorState`
  - 로딩: `Skeleton`
- **Alert/Warning 카드 규칙**
  - 넓은 빨강 배경 금지 → 좌측 스트립/아이콘/텍스트로 “신호”만 준다
  - 문구는 “원인”보다 “다음 행동(재시도/확인)” 중심
- **토스트 규칙(문장 스타일)**
  - 고객: 1줄 결과 + 1줄 다음 안내(친절)
  - 관리자: 제목은 동사형(“저장됨/변경됨”), 설명은 필요할 때만(간결)

### 3.9 UI 규칙(Interaction) — 고객/관리자 차등 적용
- **Hover(고객)**: lift 가능(아주 약하게), shadow 1단 상승
- **Hover(관리자)**: 배경/테두리만 변화(translate 금지)
- **Press**: `active:scale-[0.99]`로 고정(버튼/카드 동일 규칙)
- **Focus**: 모든 interactive 요소는 동일한 ring 규칙(현재 `.click-hint`가 기준)
- **Disabled**: opacity로만 흐리지 말고, 텍스트 대비 유지 + 커서/상태를 명확히
- **Reduced motion**: `prefers-reduced-motion` 자동 준수(고객 모션도 즉시 비활성)

### 3.10 UI 규칙(Accessibility) — 규칙으로 품질 고정
- **터치 타겟**: 최소 44x44px(현재 글로벌 규칙 유지)
- **포커스 링**: 대비가 충분한 색/두께로 통일(카드/버튼/링크 동일)
- **색으로만 의미 전달 금지**: 상태는 아이콘/라벨/텍스트를 반드시 동반

## 3.99 통일성(Consistency) 운영 규칙 — “세련됨은 규칙에서 나온다”
> 디자이너/개발자가 늘어나도 흔들리지 않게, ‘허용되는 패턴’을 문서로 고정합니다.

### 3.99.1 Single Source of Truth(SSOT)
- **색/서피스/섀도우**: `tailwind.config.ts` 토큰만 사용(임의 hex 금지)
- **상태**: `StatusPill`/`StatusBadge` + `status-meta.ts`만 사용(페이지별 커스텀 금지)
- **피드백**: `EmptyState`/`ErrorState`/`Skeleton`만 사용(임의 처리 금지)
- **클릭 힌트**: `.click-hint` 또는 카드 `interactive` 규칙만 사용(개별 컴포넌트에서 hover 남발 금지)

### 3.99.2 “시각 언어” 5가지 규칙(강제)
- **Radius**: Card `2xl`, Inset `xl`, Chip `full` (예외 금지)
- **Border**: 기본은 `border-border`(헤어라인), 강조만 `border-emphasis`(한 단계)
- **Shadow**: 기본 `shadow-card`, hover `shadow-card-hover` 2단계만
- **Spacing**: 8pt 그리드 준수(4px은 예외적으로만)
- **Color 면적**: 고채도 면적 사용 금지(색은 포인트로만)

### 3.99.3 예외 처리 프로세스(간단)
- 새로운 시각 패턴이 필요하면
  - 1) 기존 규칙으로 해결 가능한지 확인
  - 2) 해결 불가 시 “Whitelist(3.11C)”에 추가 후 프로젝트 전체에 동일 적용

## 4) 카드 컴포넌트 리팩토링 스펙(Variant 재정의)

현재 `Card variant`가 존재하므로 “스타일만” 재정의합니다.

### 4.1 공통 카드 구조(권장)
- **Header**: 타이틀/서브타이틀/우측 액션은 항상 동일한 간격
- **Body**: 인셋 박스(예: 정보/폼)는 `background-muted`로 통일
- **Footer**: CTA가 있으면 항상 하단 고정(모바일에서 스크롤로 밀리지 않게)

### 4.2 Variant별 목표
- **default**: 뉴트럴 + 최소 테두리(가장 조용)
- **info**: “설명/안내”용. 넓은 색면 금지, **아이콘 타일 + 헤어라인**으로 정보성을 부여
- **cta**: 행동 유도. 배경은 뉴트럴, 대신 **상단 헤어라인 + 은은한 wash + hover lift**
- **alert**: 경고/오류. 빨강 면적 최소화, **좌측 3px 스트립 + 텍스트 강조** 중심
- **muted**: 로딩/빈 상태. border 약화 + 배경 살짝 톤다운(“비활성 영역” 느낌)

### 4.3 “촌스러움 제거”를 위한 핵심 디테일(카드 레벨)
- **Border**: 굵은 테두리 금지 → `border`는 헤어라인 수준(불투명도 낮게)
- **Shadow**: 기본은 `shadow-card` / hover만 `shadow-card-hover`로 상승
- **Radius**: 카드 `rounded-2xl` 유지(이미 설정됨), 내부 박스는 `rounded-xl`
- **Icon tile**: 단색 배경 타일보다는 **뉴트럴 타일 + 컬러는 아이콘**에 집중

---

## 5) 동적 컴포넌트/전환(모션) 시스템

### 5.1 모션 토큰(프로덕션 규칙)
- **Duration**
  - micro(hover/press): 120–180ms
  - transition(섹션/카드 진입): 240–360ms
- **Easing**
  - 기본: `cubic-bezier(0.22, 1, 0.36, 1)` (부드러운 easeOut)
  - press: `ease-out` 짧게
- **Distance**
  - lift: 2–4px
  - fade+slide: y 8–14px 범위

### 5.2 고객 페이지(강화)
- **카드 hover/lift**: `hover:-translate-y-0.5` 수준(과한 튕김 금지)
- **press feedback**: `active:scale-[0.99]` 고정(버튼과 일관)
- **리스트 진입**: 카드 리스트는 “stagger 40ms” 정도로만(과한 모션 금지)
- **전환 적용 범위**: 홈/가이드/주문/도움말의 상단 Hero·주요 카드만 제한 적용

### 5.3 관리자 페이지(최소)
- hover는 배경/테두리만 미세하게, 이동(translate) 금지
- 전환 애니메이션은 페이지 전체에 적용하지 않고, **필요 시 Skeleton → 실데이터 교체** 정도만

### 5.4 “동적 컴포넌트” 우선순위(카드 품질 체감이 큰 것)
- **상태 변화가 있는 카드**: 주문 상태/체크인/공지(읽음 처리 등)  
  → 상태 변화 시 *색 변화 + 짧은 fade*로만 피드백
- **Inspector/Drawer/Sheet**: 헤더 고정 + 본문만 스크롤 + 오픈/클로즈 fade/slide 규칙 고정

## 6) 구현(리팩토링) 설계안 — 최소 변경으로 큰 체감 만들기

### 6.1 P0: 카드의 “표면/섀도우/테두리” 규칙 확정(가장 체감 큼)
- `components/ui/card.tsx`에서 variant 클래스 재정의(면: 뉴트럴 / 포인트: 헤어라인)
- `shadow-*` 토큰을 카드에 적극 사용(`shadow-card`, `shadow-card-hover`)
- `.click-hint`를 카드 interactive 표준으로 흡수(중복 제거)

### 6.2 P1: 아이콘 타일/헤더 패턴 컴포넌트화(촌스러움 급감)
- `CardTitle` 옆 아이콘 타일을 매번 직접 만들지 말고,
  - `CardIconBadge`(예: 32x32, 뉴트럴 배경, 아이콘 컬러만 상태색)로 통일
- 고객 QuickAction/Info 카드/공지 카드에 동일 적용

### 6.3 P1: 고객 전용 모션 래퍼(규칙을 코드로 고정)
- `GuestMotionCard` 같은 얇은 래퍼를 두어
  - hover/press/입장 모션을 “선택적으로” 부여
  - `prefers-reduced-motion` 자동 준수

### 6.4 P2: 고객 페이지 전환(탭/페이지) 최소 적용
- 하단 탭 전환 시: **fade 120ms + y 6px** 정도로만
- Scroll 진입 애니메이션은 주요 섹션 2~3개로 제한

## 7) 카드 “촌스러움 제거” 체크리스트(리뷰 기준)

- **색 면적**: 브랜드 크림/베이지가 “큰 면”을 차지하지 않는가?
- **그림자**: 기본은 약하고 hover에서만 상승하는가?
- **테두리**: 헤어라인 수준인가? (두껍게 보이면 실패)
- **아이콘 타일**: 채도 높은 배경을 남발하지 않는가?
- **텍스트 위계**: 타이틀/핵심/보조/메타가 1초 내 스캔되는가?
- **모션 일관성**: 고객은 부드럽고 짧게, 관리자는 최소인가?
- **접근성**: focus ring이 모든 interactive 카드에 적용되는가?
