# 고객 안내 페이지 진단 및 개선안

## 📋 문서 개요

**작성일**: 2024-01-XX  
**목적**: 고객 안내 페이지의 현재 상태를 진단하고, 관리자 구두 설명 리소스를 줄이기 위한 세심하고 상세한 가이드 제공을 위한 개선안 제시  
**대상 페이지**: `/guest/[token]/guide`

---

## 🎯 프로젝트 목적 재확인

1. **고객에게 세심하고 상세한 가이드 전달**
   - 숙박 이용에 필요한 모든 정보를 명확하게 제공
   - 고객이 스스로 문제를 해결할 수 있도록 충분한 정보 제공

2. **관리자 구두 설명 리소스 절감**
   - 체크인 시 반복되는 설명을 디지털 가이드로 대체
   - 고객 문의 전화/메시지 감소
   - 24시간 접근 가능한 자가 서비스 제공

3. **숙박 이용 안내 목적 달성**
   - 시설 이용 방법
   - 안전 수칙
   - 편의시설 사용법
   - 문제 해결 가이드

## 🚨 주요 문제점

### 1. **정보의 불충분성**

#### 예시: "난방/에어컨 사용법"
**현재 내용**: "컨트롤러를 사용하여 원하는 온도를 설정하세요. 권장 온도는 20-22도입니다."

**문제점**:
- 컨트롤러 위치가 어디인지 모름
- 컨트롤러 사진/이미지 없음
- 버튼 설명 없음
- 모드 전환 방법 없음
- 작동하지 않을 때 대처 방법 없음
- 전력 소비 관련 안내 없음

**관리자가 실제로 설명하는 내용** (추정):
- 컨트롤러 위치 (벽면 또는 리모컨)
- 전원 켜기/끄기 방법
- 냉방/난방 모드 전환
- 온도 설정 방법
- 타이머 설정
- 작동하지 않을 때 체크리스트
- 전력 절약 팁

### 2. **시각적 자료 부족**

현재 대부분의 가이드 항목에 이미지가 없어, 텍스트만으로는 이해하기 어려움:
- 컨트롤러 사진
- 시설 위치 사진
- 단계별 스크린샷
- 다이어그램

### 3. **문제 해결 가이드 부족**

고객이 문제를 겪었을 때:
- 원인 파악 방법
- 자가 해결 방법
- 관리자 연락 시기
- 응급 조치 방법

### 4. **체크리스트 기능 부족**

체크인/체크아웃 시 확인해야 할 사항들을 체크리스트로 제공하지 않음:
- 체크인 시 확인 사항
- 체크아웃 시 확인 사항
- 시설 점검 체크리스트

### 5. **접근성 및 사용성**

- 모바일 최적화는 되어 있으나, 긴 텍스트 가독성 개선 필요
- 키보드 네비게이션 지원 부족
- 스크린 리더 최적화 부족

## 💡 개선안

### Phase 1: 콘텐츠 확장 및 상세화 (우선순위: 높음)

#### 1.1 가이드 항목 확장

**추가 필요 가이드 항목**:

1. **체크인 가이드**
   - 체크인 절차
   - 객실 위치 안내
   - 키/도어락 사용법
   - 초기 점검 사항

2. **체크아웃 가이드**
   - 체크아웃 절차
   - 정리 사항 체크리스트
   - 키 반납 방법
   - 후기 작성 안내

3. **시설 이용 가이드**
   - 샤워실 사용법 (온수, 배수)
   - 화장실 사용법
   - 주방 시설 사용법
   - 세탁기 사용법 (있는 경우)
   - 야외 테라스/정원 이용 안내

4. **안전 가이드**
   - 화재 안전 수칙
   - 응급 연락처
   - 의료 시설 위치
   - 비상 대피 경로

5. **주변 정보**
   - 인근 편의시설 (마트, 약국, 병원)
   - 관광지 정보
   - 맛집 추천
   - 교통 정보

6. **규칙 및 에티켓**
   - 소음 규칙
   - 흡연 규칙
   - 반려동물 규칙
   - 공용 공간 이용 규칙

7. **문제 해결 가이드**
   - WiFi 연결 안 될 때
   - 온수가 안 나올 때
   - 난방/에어컨 작동 안 할 때
   - 배수 막힘
   - 전기 차단
   - 기타 장비 고장

#### 1.2 기존 가이드 항목 상세화

각 가이드 항목을 다음 구조로 확장:

```typescript
interface GuideItem {
  id: string;
  category: string;
  title: string;
  // 기존 필드
  content: string;
  images: string[];
  warning: boolean;
  warningText?: string;
  
  // 추가 필드
  overview: string; // 간단한 개요
  steps: Step[]; // 단계별 가이드
  faq: FAQ[]; // 자주 묻는 질문
  troubleshooting: Troubleshooting[]; // 문제 해결
  tips: string[]; // 유용한 팁
  relatedGuides: string[]; // 관련 가이드 ID
  videoUrl?: string; // 비디오 가이드 URL
  checklist?: ChecklistItem[]; // 체크리스트
}

interface Step {
  number: number;
  title: string;
  description: string;
  image?: string;
  warning?: string;
}

interface FAQ {
  question: string;
  answer: string;
}

interface Troubleshooting {
  problem: string;
  solution: string;
  requiresContact: boolean; // 관리자 연락 필요 여부
}

interface ChecklistItem {
  id: string;
  text: string;
  required: boolean; // 필수 확인 사항 여부
}
```

### Phase 2: UI/UX 개선 (우선순위: 높음)

#### 2.1 가이드 표시 방식 개선

**현재**: 아코디언 스타일 (텍스트만)

**개선안**: 
- **카드 기반 레이아웃**: 각 가이드 항목을 카드로 표시
- **프리뷰 이미지**: 각 가이드 항목에 대표 이미지 표시
- **진행 상태 표시**: 체크리스트 완료 여부 표시
- **즐겨찾기 기능**: 자주 보는 가이드 즐겨찾기

#### 2.2 단계별 가이드 UI

BBQ 가이드처럼 단계별 가이드를 모든 가이드 항목에 적용:

```tsx
<StepByStepGuide 
  steps={guideItem.steps}
  currentStep={currentStep}
  onStepComplete={(step) => markStepComplete(step)}
/>
```

**기능**:
- 단계별 진행 표시
- 각 단계 완료 체크
- 이전/다음 단계 네비게이션
- 진행률 표시

#### 2.3 체크리스트 UI

체크인/체크아웃 및 시설 점검용 체크리스트:

```tsx
<Checklist 
  items={guideItem.checklist}
  onComplete={(completedItems) => saveProgress(completedItems)}
  showProgress={true}
/>
```

**기능**:
- 체크박스로 항목 체크
- 진행률 표시
- 로컬 스토리지에 저장 (페이지 새로고침 시에도 유지)
- 필수 항목 강조

#### 2.4 FAQ 섹션 추가

각 가이드 항목 하단에 FAQ 섹션:

```tsx
<FAQSection 
  faqs={guideItem.faq}
  searchable={true}
/>
```

**기능**:
- 아코디언 스타일 FAQ
- 검색 가능
- 관련 가이드 링크

#### 2.5 문제 해결 가이드 UI

트러블슈팅 섹션:

```tsx
<TroubleshootingSection 
  items={guideItem.troubleshooting}
  onContactManager={() => openContactModal()}
/>
```

**기능**:
- 문제별 해결 방법 표시
- 관리자 연락 필요 시 즉시 연락 버튼
- 단계별 해결 가이드

### Phase 3: 검색 및 네비게이션 개선 (우선순위: 중간)

#### 3.1 검색 기능 고도화

**현재**: 단순 키워드 매칭

**개선안**:

- **자동완성**: 검색어 입력 시 자동완성 제안
- **검색 결과 하이라이트**: 검색어 하이라이트
- **최근 검색어**: 최근 검색어 저장 및 표시

#### 3.2 빠른 접근 섹션

#### 4.1 이미지 추가

모든 가이드 항목에 최소 1개 이상의 이미지 추가:
- 컨트롤러/장비 사진
- 시설 위치 사진
- 단계별 스크린샷
- 다이어그램

#### 4.2 비디오 가이드

복잡한 가이드 항목에 비디오 추가:
- BBQ/불멍 사용법 (우선)
- 빔 프로젝터 연결 방법
- 체크인/체크아웃 절차

**비디오 플레이어 UI**:
```tsx
<VideoGuide 
  videoUrl={guideItem.videoUrl}
  transcript={guideItem.transcript} // 자막
  chapters={guideItem.videoChapters} // 챕터
/>
```

#### 4.3 인터랙티브 다이어그램

시설 배치도, 전기 배선도 등을 인터랙티브하게 표시:

```tsx
<InteractiveDiagram 
  diagram={guideItem.diagram}
  hotspots={guideItem.hotspots} // 클릭 가능한 영역
/>
```

// lib/types/guide.ts

export interface GuideItem {
  id: string;
  category: string;
  title: string;
  overview: string; // 간단한 개요 (1-2줄)
  content: string; // 상세 설명

  videoUrl?: string;
  warning: boolean;
  warningText?: string;
  steps?: Step[];
  faq?: FAQ[];
  troubleshooting?: Troubleshooting[];
  tips?: string[];

  checklist?: ChecklistItem[];
  tags?: string[]; // 검색용 태그
  priority?: number; // 우선순위 (높을수록 상단 노출)

}

export interface Step {
  number: number;
  title: string;
  description: string;
  image?: string;
  warning?: string;
  estimatedTime?: string; // 예상 소요 시간
}

export interface FAQ {
  id: string;
  question: string;
  answer: string;
  relatedGuideId?: string;
}

export interface Troubleshooting {
  id: string;
  problem: string;
  solution: string;

  contactMethod?: 'phone' | 'message' | 'email';
  estimatedTime?: string; // 예상 해결 시간
}

export interface ChecklistItem {
  id: string;
  text: string;
  required: boolean;
  category?: string; // 'checkin' | 'checkout' | 'facility'
  helpText?: string; // 도움말
}

## 📌 결론

현재 안내 페이지는 기본적인 구조는 갖추고 있으나, **관리자 구두 설명을 대체하기에는 콘텐츠의 상세도와 완성도가 부족**합니다.

**핵심 개선 방향**:
1. ✅ **콘텐츠 확장 및 상세화**: 단계별 가이드, FAQ, 문제 해결 가이드 추가
2. ✅ **시각적 자료 보강**: 이미지, 비디오, 다이어그램 추가
3. ✅ **인터랙티브 요소 추가**: 체크리스트, 단계별 진행 추적
4. ✅ **검색 기능 고도화**: 동의어 검색, 자동완성
5. ✅ **접근성 개선**: ARIA, 키보드 네비게이션

위 개선안을 단계적으로 구현하면, 고객이 스스로 문제를 해결할 수 있는 **세심하고 상세한 디지털 가이드**를 제공할 수 있으며, 이를 통해 **관리자의 구두 설명 리소스를 크게 줄일 수 있습니다**.
