# 올바른 경로 찾기 가이드

## 🔍 문제

경로가 존재하지 않는다는 오류가 발생했습니다. 실제 경로를 확인해야 합니다.

## ✅ 해결 방법

### 방법 1: Desktop 폴더 내용 확인

PowerShell에서 다음 명령어를 실행하여 Desktop 폴더의 내용을 확인하세요:

```powershell
# Desktop 폴더의 모든 폴더 목록 확인
Get-ChildItem "C:\Users\김지훈\Desktop" -Directory | Select-Object Name
```

또는 간단하게:
```powershell
dir "C:\Users\김지훈\Desktop"
```

### 방법 2: 파일 탐색기에서 경로 복사

1. **파일 탐색기 열기**
2. **Desktop 폴더로 이동**
3. **`[개인] 김지훈` 폴더 찾기**
4. **폴더를 우클릭 → "경로로 복사" 또는 "경로 복사"**
5. **PowerShell에서 붙여넣기**

### 방법 3: 탭 자동완성 사용

PowerShell에서 경로를 입력하다가 `Tab` 키를 누르면 자동완성됩니다:

```powershell
cd "C:\Users\김지훈\Desktop\["  # Tab 키 누르면 자동완성
```

### 방법 4: 현재 작업 디렉토리 확인

현재 작업 디렉토리가 이미 올바른 위치일 수 있습니다:

```powershell
# 현재 경로 확인
pwd

# 또는
Get-Location
```

현재 경로가 `C:\Users\김지훈\Desktop\[개인] 김지훈\오우스 자동화`라면 이미 올바른 위치입니다!

## 📋 실제 경로 확인 방법

### 1단계: Desktop 폴더 확인

```powershell
cd "C:\Users\김지훈\Desktop"
dir
```

### 2단계: 폴더 이름 확인

Desktop에 있는 폴더 목록을 확인하고, 정확한 폴더 이름을 찾으세요.

**가능한 폴더 이름:**
- `[개인] 김지훈`
- `[개인]김지훈` (공백 없음)
- `개인 김지훈` (대괄호 없음)
- 다른 이름

### 3단계: 정확한 경로로 이동

확인한 폴더 이름을 사용하여 이동:

```powershell
# 예시 1: 공백이 있는 경우
cd "C:\Users\김지훈\Desktop\[개인] 김지훈\오우스 자동화"

# 예시 2: 공백이 없는 경우
cd "C:\Users\김지훈\Desktop\[개인]김지훈\오우스 자동화"

# 예시 3: 대괄호가 없는 경우
cd "C:\Users\김지훈\Desktop\개인 김지훈\오우스 자동화"
```

## 🔧 대안: 상대 경로 사용

현재 Desktop에 있다면:

```powershell
# Desktop에서 상대 경로로 이동
cd ".\[개인] 김지훈\오우스 자동화"
```

## 💡 빠른 해결 방법

### 파일 탐색기 사용 (가장 확실)

1. **파일 탐색기 열기**
2. **`C:\Users\김지훈\Desktop\[개인] 김지훈\오우스 자동화` 폴더로 이동**
3. **주소 표시줄 클릭 → 전체 경로 복사 (Ctrl+C)**
4. **PowerShell에서 실행:**

```powershell
cd "복사한_경로"
```

### 또는 PowerShell에서 직접 열기

```powershell
# 파일 탐색기에서 해당 폴더 열기
explorer "C:\Users\김지훈\Desktop"
```

그 다음 파일 탐색기에서 정확한 경로를 확인하세요.

## ⚠️ 주의사항

1. **대소문자 구분**: Windows는 대소문자를 구분하지 않지만, 정확한 경로를 사용하는 것이 좋습니다.
2. **공백**: 폴더 이름에 공백이 있으면 따옴표로 감싸야 합니다.
3. **특수문자**: 대괄호 `[`, `]`는 특수문자이므로 따옴표로 감싸야 합니다.
