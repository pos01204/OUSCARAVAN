## 핵심 결론(요약)

### 현재 버전의 강점(디자인/시각 관점)

- **톤 & 무드**: 라이트 테마 + 무채색 배경으로 카드 대비가 좋아 “정돈된 인상”을 줌.
- **카드 시스템**: `Card` 기본 보더/섀도/라운드가 안정적이라 전체 화면 퀄리티를 받쳐줌.
- **정보 구조**: 홈에서 공지 → Quick Action → 핵심 카드로 흐름이 명확하고, 상세는 `InfoInspector`로 2차 계층화가 가능함.

### 가장 큰 개선 포인트(고객 체감 기준)

- **상태 변화의 ‘확신’ 부족**: 주문/체크인/체크아웃처럼 중요한 행동은 “처리 중/완료/실패 후 다음 행동”이 화면 내에서 더 분명히 보여야 함.
- **시각적 위계의 강약 부족**: 중요 카드(공지/체크인아웃/주문 상태)가 다른 카드와 비슷한 무게로 보여 핵심이 섞여 보일 수 있음.
- **모션/강조 효과의 불균형**: 일부 기능만 강한 시각 효과(예: confetti)를 사용하면 전체 톤이 깨질 수 있음.

## 디자인/시각 시스템 진단

### 1) 색/대비/배경(테마)

- **현 상태**: `globals.css`에서 `--background: #F7F7F7` 기반으로 카드 대비 확보 → 방향 좋음.
- **개선**:
  - **섹션 덩어리감**: 홈처럼 카드가 많을 때 “흰 박스 나열”로 보일 수 있어, 섹션 컨테이너에 아주 약한 틴트(`bg-background-muted/30`) 또는 구분선을 규칙화 권장.
  - **강조 규칙 제한**: 강조 스타일(프리미엄/상단 라인/강한 배경)을 1~2개 규칙으로 제한해 일관성 유지.

### 2) 타이포(크기/두께/행간)

- **현 상태**: `GuestPageHeader`는 안정적. 다만 페이지 내부 섹션 타이틀/설명/본문의 굵기 규칙이 더 필요.
- **권장 규칙(제안)**:
  - **페이지 타이틀**: 24–30px / 700–800
  - **섹션 타이틀**: 18–22px / 700
  - **카드 타이틀**: 16–18px / 600–700
  - **보조 설명**: 12–14px / 400–500 (`leading-relaxed`)

### 3) 카드/보더/섀도(컴포넌트 일관성)

- **현 상태**: `Card`는 기본 스타일이 매우 좋음(보더/섀도/호버).
- **개선**:
  - **카드 내부 패딩 규칙**: `CardHeader pb-3` / `CardContent space-y-3~4` 기본값을 문서화해서 페이지별 편차를 줄이기.
  - **Interactive 사용 기준**: 카드 전체가 클릭되는 경우에만 `interactive` 사용(버튼이 존재하는 카드까지 interactive면 과잉 피드백 가능).

### 4) 아이콘/시각 요소(‘클린’ 유지)

- **현 상태**: 아이콘 수는 적정. 다만 탭/상태/라벨에 아이콘이 겹치면 “장식화”될 수 있음.
- **개선**:
  - 아이콘은 **행동(CTA)/상태(대기·준비·완료)/긴급(공지)**에 우선 배치.
  - 아이콘 칩(배경색/크기)을 페이지 전반에 통일.

### 5) 모션/애니메이션(절제된 프리미엄)

- **현 상태**: Hero 모션 + 탭 전환 + confetti 등 일부만 강함.
- **개선**:
  - 모션 레벨을 2단계로 제한: (A) 기본 페이드/슬라이드(150~250ms), (B) 축하/강조(아주 제한적).
  - `prefers-reduced-motion` 원칙을 confetti 외에도 적용.

## 상태/피드백(로딩·성공·실패) 진단

### 핵심 원칙

1) **지금 무슨 일이 일어나는지**(처리 중)  
2) **결과가 반영됐는지**(상태/배지/시간)  
3) **실패 시 다음 행동이 뭔지**(재시도/대안)

### 현재 진단

- 토스트는 있으나, **화면 내(inline) 피드백**이 약한 곳이 있음(주문 상태, 체크인/아웃).
- `useGuestOrders`는 마운트 1회 조회라서 “갱신이 안 된다”는 인상을 줄 수 있음.

### 개선 제안(디자인+기능 결합)

- 주문/체크인/공지에 **“배지 + 타임스탬프 + 새로고침(또는 자동 갱신)”** 최소 세트 제공.
- 에러 상태는 “짧은 문구 + 재시도 버튼(1차)” 패턴 통일.
- 로딩은 스켈레톤 + 짧은 안내 문구 조합 통일.

## 전체 개요

### 현재 상태 요약

**강점**:
- ✅ 명확한 네비게이션 구조 (하단 네비게이션, Quick Actions)
- ✅ 일관된 디자인 시스템 (카드, 색상, 타이포그래피)
- ✅ 모바일 최적화된 레이아웃
- ✅ 실시간 주문 동기화 (`GuestOrderSync`)
- ✅ 공지사항 시스템 (읽음/안 읽음 상태 관리)
- ✅ 체크아웃 알림 설정 (PWA 기반)

**개선 필요 영역**:
- ⚠️ 주문 상태 실시간 업데이트 부족
- ⚠️ 오프라인 상태 처리 미흡
- ⚠️ 에러 복구 메커니즘 부족
- ⚠️ 사용자 피드백 부족 (로딩, 성공, 실패)
- ⚠️ 접근성 개선 여지 (키보드 네비게이션, 스크린 리더)
- ⚠️ 데이터 새로고침 수동 의존

## 페이지별 진단

### 1. 홈 페이지 (`/guest/[token]`)

#### 현재 기능
- Hero 섹션 (환영 메시지)
- 공지사항 (긴급/일반, 읽음 상태)
- Quick Actions (체크인/아웃, WiFi, 주문, 도움말)
- 약도 카드 (배정된 공간 표시)
- WiFi/시간/체크인아웃 카드
- 체크아웃 리마인더
- 최근 주문 요약

#### 개선 사항

**🔴 높은 우선순위**

1. **주문 상태 실시간 업데이트 부족**
   - 현재: `GuestOrderSync`가 백그라운드에서 주기적으로 동기화하지만, UI에 실시간 반영이 즉각적이지 않음
   - 문제: 고객이 주문 상태 변경을 즉시 확인하기 어려움
   - 개선: WebSocket 또는 Server-Sent Events (SSE)로 주문 상태 변경 시 즉시 알림
   - 위치: `components/guest/GuestOrderSync.tsx`, `components/guest/RecentOrdersSummary.tsx`

2. **공지사항 읽음 상태 영구 저장 부족**
   - 현재: `localStorage`에 저장되지만, 브라우저 데이터 삭제 시 초기화됨
   - 문제: 고객이 다른 기기에서 접속하거나 브라우저 데이터를 삭제하면 다시 읽지 않은 공지로 표시됨
   - 개선: 서버에 읽음 상태 저장 (백엔드 API 추가 필요)
   - 위치: `components/guest/GuestAnnouncements.tsx`

3. **체크아웃 리마인더 알림 신뢰성**
   - 현재: PWA Service Worker 기반이지만, 브라우저별 지원 차이 및 알림 권한 필요
   - 문제: 일부 환경에서 알림이 작동하지 않을 수 있음
   - 개선: 알림 설정 실패 시 대체 방법 제공 (예: 페이지 접속 시 배너 표시)
   - 위치: `components/features/CheckoutReminder.tsx`

**🟡 중간 우선순위**

4. **약도 카드 접근성**
   - 현재: SVG 기반 약도로 시각적 정보만 제공
   - 문제: 스크린 리더 사용자나 저시력 사용자가 정보를 파악하기 어려움
   - 개선: ARIA 레이블 추가, 텍스트 기반 설명 제공
   - 위치: `components/features/FloorPlanSVG.tsx`, `components/guest/FloorPlanCard.tsx`

5. **최근 주문 요약 상세 정보 부족**
   - 현재: 주문명, 금액, 상태만 표시
   - 문제: 배송 시간, 특이사항 등 추가 정보 확인을 위해 전체 보기로 이동해야 함
   - 개선: 카드 클릭 시 상세 정보 Drawer/Sheet 표시
   - 위치: `components/guest/RecentOrdersSummary.tsx`

6. **로딩 상태 일관성**
   - 현재: 각 컴포넌트별로 개별 로딩 상태 관리
   - 문제: 초기 로딩 시 여러 스켈레톤이 동시에 표시되어 시각적 혼란
   - 개선: 페이지 레벨 로딩 상태 통합 관리
   - 위치: `app/guest/[token]/page.tsx`

**🟢 낮은 우선순위**

7. **Hero 섹션 브랜드 미디어 최적화**
   - 현재: 이미지/비디오가 있으면 표시, 없으면 그라데이션 배경
   - 개선: 이미지 로딩 실패 시 대체 텍스트 또는 기본 이미지 제공
   - 위치: `components/guest/BrandMediaLayer.tsx`

### 2. 주문/카페 페이지 (`/guest/[token]/order`)

#### 현재 기능
- 쿠폰 카드 (플립 애니메이션)
- 탭 네비게이션 (주문/카페)
- 배송 주문 (불멍/바베큐, 키오스크)
- 주문 상태 요약 바
- 주문 내역 (필터링, 상세 보기, 재주문)

#### 개선 사항

**🔴 높은 우선순위**

1. **주문 상태 실시간 업데이트 부족**
   - 현재: `useGuestOrders`가 마운트 시 한 번만 조회, 수동 새로고침 필요
   - 문제: 주문 상태 변경 시 고객이 즉시 확인 불가
   - 개선: 
     - 주문 상태 변경 시 자동 새로고침 (폴링 간격 단축 또는 SSE)
     - 주문 상태 변경 시 토스트 알림
   - 위치: `lib/hooks/useGuestOrders.ts`, `components/guest/GuestOrderContent.tsx`

2. **주문 폼 단계별 진행 피드백 부족**
   - 현재: 단계별 진행은 있지만, 현재 단계와 남은 단계에 대한 명확한 표시 부족
   - 문제: 고객이 주문 진행 상황을 파악하기 어려움
   - 개선: 진행 바(Progress Bar) 또는 스텝 인디케이터 추가
   - 위치: `components/features/OrderForm.tsx`, `components/features/KioskOrderForm.tsx`

3. **주문 실패 시 복구 메커니즘 부족**
   - 현재: 에러 발생 시 토스트만 표시, 재시도 버튼 없음
   - 문제: 네트워크 오류 등 일시적 문제 시 재시도 불가
   - 개선: 에러 발생 시 "다시 시도" 버튼 제공, 자동 재시도 (exponential backoff)
   - 위치: `components/features/OrderForm.tsx`, `components/features/KioskOrderForm.tsx`

**🟡 중간 우선순위**

4. **주문 내역 필터링 UX 개선**
   - 현재: 상태별 필터링은 있지만, 날짜/시간 기준 필터링 없음
   - 문제: 주문이 많을 때 특정 기간의 주문을 찾기 어려움
   - 개선: 날짜 범위 필터 추가, 검색 기능 추가
   - 위치: `components/features/OrderHistory.tsx`, `components/guest/GuestOrderContent.tsx`

5. **주문 상세 정보 표시 개선**
   - 현재: `InfoInspector`로 상세 정보 표시하지만, 배송 시간, 특이사항 등이 명확하지 않을 수 있음
   - 개선: 주문 상세 정보 구조화 (항목별 섹션 구분)
   - 위치: `components/features/OrderHistory.tsx`

6. **재주문 기능 개선**
   - 현재: 재주문 시 이전 주문 정보를 폼에 채워넣지만, 수정 후 주문하는 과정이 명확하지 않음
   - 개선: 재주문 시 "이전 주문과 동일하게 주문" 옵션 제공
   - 위치: `components/features/OrderHistory.tsx`, `components/guest/GuestOrderContent.tsx`

**🟢 낮은 우선순위**

7. **쿠폰 카드 플립 애니메이션 접근성**
   - 현재: 플립 애니메이션이 있지만, 키보드 접근성 부족
   - 개선: 키보드로도 플립 가능하도록 개선 (Space/Enter 키)
   - 위치: `components/features/CouponFlip.tsx`

8. **카페 이용 탭 정보 확장**
   - 현재: 기본 정보만 표시
   - 개선: 카페 메뉴, 운영 시간, 할인 정보 등 추가
   - 위치: `components/features/CafeInfoTab.tsx`

### 3. 가이드 페이지 (`/guest/[token]/guide`)

#### 현재 기능

**🔴 높은 우선순위**

   - 현재: 검색 결과만 필터링하여 표시
   - 문제: 검색어가 결과에서 강조되지 않아 찾기 어려움
   - 개선: 검색어 하이라이트 (텍스트 강조)

## 권장 실행 순서(현실적인 롤아웃)

### Phase 0 (반나절~1일): ‘보이는 개선’ 빠른 승리(디자인/피드백)

- **검색 하이라이트**: 가이드/도움말에서 검색어가 결과 텍스트에 강조되어 “찾는 느낌”을 강화
- **에러 패턴 통일**: 에러 카드에 “재시도” 버튼 1개를 기본 제공(주문/최근주문/체크인아웃)
- **상태 배지 통일**: 대기/준비/완료(3단계) 색/문구 규칙을 홈 요약/주문 내역/상세에서 동일하게
- **홈 스크롤 피로 감소**: 홈 하단(최근 주문/리마인더)은 요약/접기 기본값으로 전환

### Phase 1 (2~4일): 고객 만족도 영향 큰 핵심(실시간/확신)

- **주문 상태 자동 갱신**: 폴링 개선(간격/백오프) 또는 가능하면 SSE로 “상태 변화를 바로 보이게”
- **체크인/체크아웃 서버 동기화**: 새로고침/기기 변경에도 상태가 유지되도록 신뢰성 강화
- **주문 완료 인라인 피드백**: 토스트만이 아니라 “주문 접수됨(요약 카드)”을 1회 보여 확신 제공

### Phase 2 (1~2주): 프리미엄 완성도(기억/지속/안정)

- **공지 읽음/즐겨찾기**: 단말/브라우저를 넘어 상태가 유지되도록 서버 저장(가능하면)
- **접근성 최소 기준**: 포커스 링/키보드 이동/대비를 WCAG AA에 근접하도록 정리
- **네트워크 불안정 대응**: 오프라인/지연 시 UX(대기열/안내/캐싱) 강화

## 결론

현재 고객 페이지는 **기본적인 기능과 사용성**은 양호하고, 특히 **카드 기반 시각 시스템**이 이미 탄탄합니다. 다음 단계는 “더 예쁘게”보다는, 고객이 가장 민감하게 느끼는 **상태 변화의 확신(주문/체크인/공지)**과 **정보 위계(중요 카드가 먼저 보이는 구조)**를 강화하는 것입니다.  
즉, **디자인(시각)과 기능(실시간/복구)이 맞물린 부분**을 우선 개선하면 체감 퀄리티가 크게 상승합니다.

또한, **검색 결과 하이라이트**와 **에러 재시도 버튼** 같은 작은 개선사항들도 사용자 경험을 크게 향상시킬 수 있으므로 즉시 적용 가능한 항목부터 진행하는 것이 좋습니다.
