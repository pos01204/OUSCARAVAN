## 고객 가이드 페이지(Guest Guide) 럭셔리 업그레이드 제안서 (목업 포함)

- **대상**: `app/guest/[token]/guide` (고객 “안내/가이드”)
- **디자인 컨셉**: **고급 호텔 컨시어지(Quiet Luxury)**  
  - 색보다 **레이어/여백/타이포/규칙**으로 고급스러움 구현  
  - “필요한 순간에만 강해지는” 강조(주의/긴급/CTA)
- **목표**:
  - 고객이 “지금 당장 필요한 정보”를 5초 내 찾기
  - 가이드 UI의 **완성도(정돈감/일관성)** 강화
  - 콘텐츠가 늘어도 **복잡해 보이지 않는 구조** 확보

---

## 1) 현재 페이지에서 흔히 발생하는 완성도 저하 원인(진단 기준)

- **중첩 카드가 많아 보임**: 카드 안에 카드가 계속 생기면 깊이가 “싸구려”처럼 보임
- **행동(CTA) 규칙이 섞임**: 어떤 곳은 상세, 어떤 곳은 복사, 어떤 곳은 전화… 시각 규칙이 통일되지 않으면 복잡해짐
- **이미지의 규격/톤 불일치**: 비율/둥근 정도/캡션/품질이 섞이면 페이지 완성도가 급락
- **Step(절차)의 정보 밀도 불균형**: 어떤 단계는 길고 어떤 단계는 짧으면 스캔이 어려움
- **중요 정보(체크인/WiFi)가 리스트 속에 묻힘**: “Today / Quick actions”가 없으면 발견성이 떨어짐

---

## 2) 고급 호텔 톤을 만드는 디자인 방법(규칙)

### 2.1 레이어(Depth) 규칙
- **Page**: `bg-neutral-50`
- **Section surface(섹션 컨테이너)**: `bg-white/70 backdrop-blur border border-neutral-200/70 shadow-[0_8px_24px_rgba(0,0,0,0.06)]`
- **Card(리스트 아이템)**: 섹션 내부에서는 **그림자 금지 또는 매우 약하게**, `border` 중심
- **Callout(주의/팁)**: 컬러를 쓰더라도 **초저채도 틴트**만 (`amber-50/40`, `blue-50/40`)

### 2.2 타이포 위계(고급스러운 느낌은 “굵기”보다 “리듬”)
- **H1**: `text-2xl font-semibold tracking-tight`
- **Section title**: `text-base font-semibold tracking-tight`
- **Body**: `text-sm text-neutral-600 leading-relaxed`
- **Meta(캡션/라벨)**: `text-[11px] text-neutral-400 tracking-[0.18em] uppercase`

### 2.3 이미지 규격화(완성도 체감 최상)
- **비율 통일**: 기본 4:3 (또는 16:9 중 하나로 고정)
- **스타일 통일**: `rounded-2xl border border-neutral-200/70`
- **캡션 필수**: “무엇을 찍은 사진인지/어디에 있는지” 1줄

---

## 3) 정보 구조(IA) 재설계: “오늘 필요한 것”을 먼저

### 3.1 페이지 섹션 구성(권장)
1. **Hero / Summary**
2. **Today’s Essentials(오늘의 필수) 2~4개**
3. **카테고리 + 전체 가이드 목록**
4. (옵션) **검색**은 “간단”만 (키워드 1개 입력)

### 3.2 카테고리(업무 흐름 기준)
- **체크인/체크아웃**
- **실내(난방, 온수, 와이파이, 프로젝터)**
- **요리(BBQ/불멍)**
- **규칙(주차, 쓰레기)**
- **편의(주변시설/문의)**

## 4) 화면 목업(ASCII)

### 4.1 가이드 메인(목록) — “호텔 컨시어지 홈” 느낌

```
┌──────────────────────────────────────┐
│ OUSCARAVAN                           │
├──────────────────────────────────────┤
│ 안내                                 │
│ 오늘 필요한 정보를 빠르게 확인하세요  │
│                                      │
│ [ 오늘의 필수 ]                       │  ← 섹션 컨테이너(1)
│ ┌──────────────────────────────────┐ │
│ │  WiFi 연결          (복사/QR)     │ │  ← Quick Action Card
│ │  체크인/체크아웃     (시간/규정)   │ │
│ │  BBQ/불멍            (주문/안전)   │ │
│ │  쓰레기 분리수거      (종류/위치)  │ │
│ └──────────────────────────────────┘ │
│                                      │
│ [ 카테고리 ]                          │  ← 섹션 컨테이너(2)
│  (전체) (체크인) (실내) (요리) (규칙)  │  ← pill
│                                      │
│ [ 전체 가이드 ]                        │  ← 섹션 컨테이너(3)
│ ┌──────────────────────────────────┐ │
│ │ 난방/에어컨 사용법     1~2줄 요약 │ │  ← GuideCard
│ │ 온수 사용 안내         1~2줄 요약 │ │
│ │ 빔 프로젝터 연결        1~2줄 요약 │ │
│ │ ...                                  │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

**핵심**: 위에서 아래로 “필수 → 탐색” 흐름이 명확해져 복잡도가 급감합니다.

### 4.2 Quick Action(예: WiFi) — 바텀시트/모달(즉시 해결)

```
┌──────────────────────────────────────┐
│ WiFi                                 │
│ OUS_Guest                            │
│ 비밀번호: (없음)                     │
│                                      │
│ [ 비밀번호 복사 ]   [ QR 보기 ]       │
│                                      │
│ TIP: 연결이 안 되면 WiFi 재시도 후    │
│      그래도 안 되면 관리자에게 문의   │
└──────────────────────────────────────┘
```

**고급스럽게 보이려면**: 정보량이 많아도 “행동 버튼 1~2개 + 짧은 팁”으로 끝내기.

### 4.3 가이드 상세(절차 중심) — “요약 → 단계 → 팁/주의”

```
┌──────────────────────────────────────┐
│ 난방/에어컨 사용법                    │
│ 컨트롤러를 사용해 온도를 설정합니다   │
│                                      │
│ [ 요약 ]                              │  ← compact meta
│ 권장 온도 20~22도 / 설정 후 10분 대기 │
│                                      │
│ [ 단계 ]                              │  ← Stepper (스캔 용이)
│ ① 컨트롤러 위치 확인                  │
│    (이미지 4:3 + 캡션)                │
│ ② 전원 켜기                           │
│ ③ 모드 선택                           │
│ ④ 온도 설정(권장 20~22도)             │
│ ⑤ 풍량 조절                           │
│                                      │
│ [ 자주 묻는 문제 ]                     │  ← Troubleshooting (압축)
│ ▸ 전원이 안 켜져요                     │
│ ▸ 온도가 안 올라가요                   │
│                                      │
│ [ 관련 바로가기 ]                      │
│  관리자 연락 / 도움말(FAQ) / 주문하기  │
└──────────────────────────────────────┘
```

## 5) 컴포넌트 설계(“완성도”를 만드는 조립 방식)

### 5.1 공통 Shell
- **`GuidePageShell`**
  - 페이지 배경/최대폭/섹션 간격 표준화
  - iOS safe area 고려

### 5.2 상단 영역
- **`GuideHero`**
  - 제목 + 1줄 설명(불필요한 장식 제거)
  - 필요 시 “오늘 날짜/체크인 상태” 같은 최소 컨텍스트

### 5.3 오늘의 필수(Quick Actions)
- **`EssentialsGrid`**
  - 2열 그리드(모바일), CTA 규칙 통일
  - 액션 타입: `copy | qr | navigate | sheet`
- **`QuickActionSheet`**
  - WiFi/체크인/BBQ/쓰레기 같은 “즉시 해결”을 바텀시트로 제공

### 5.4 목록(탐색)
- **`CategoryPills`**
  - 스크롤 가능한 pill, “선택 상태”만 선명하게
- **`GuideCard`**
  - 타이틀/요약/메타(소요시간, 중요도)
  - hover/press는 150ms, prefers-reduced-motion 대응

### 5.5 상세(절차)
- **`GuideDetailHeader`**
- **`StepList` / `StepItem`**
  - Step 번호 배지(고급스럽게: 작은 원형, neutral)
- **`MediaFrame`**
  - 이미지 비율/보더/캡션 통일
- **`Callout`**
  - `variant: tip | note | warning` (색은 초저채도)
- **`TroubleshootingAccordion`**
  - 문제→해결 요약 + 필요 시 연락 CTA

## 6) 동적 효과/상태(난이도 높지만 체감 품질 큼)

### 6.1 “읽던 위치/열어본 항목” 저장
- 마지막 본 가이드 ID, 펼친 troubleshooting, 스크롤 위치를 저장
- 재방문 시 “이어서 보기” 버튼 제공(상단에 작게)

### 6.2 개인화(예약 토큰 기반 컨텍스트)
- 체크인 당일: “체크인/주차/와이파이” 우선
- 체크아웃 당일: “체크아웃/쓰레기/정리” 우선

### 6.3 오프라인/저신뢰 네트워크 대비
- 핵심 텍스트(steps/FAQ)는 로컬 캐시 가능
- 이미지 실패 시 캡션/텍스트로 충분히 이해되게 설계

### 6.4 이벤트 로깅(운영 품질)
- `guide_view`, `guide_open`, `guide_step_expand`, `guide_quick_action`, `guide_copy_wifi`, `guide_open_qr`
- 고객이 “어디에서 막히는지” 추적 가능 → 콘텐츠 개선 루프 형성

## 7) 출시 수준 체크리스트(디자인 완성도)

- **레이아웃**
  - 섹션 컨테이너 3~4개로 명확히 구분
  - 중첩 카드 최소화(카드 안에 카드 금지)
- **타이포**
  - 제목/설명/캡션 위계가 일정
- **이미지**
  - 비율 통일 + 캡션 필수
- **행동(CTA)**
  - 버튼 규칙 통일(Primary 1개, Secondary 1개)
  - 복사/QR/전화 등 행동은 같은 위치에 반복
- **상태**
  - 로딩 시 skeleton, 실패 시 retry
  - 이전 상태 복원(이어보기)

## 8) 구현 로드맵(추천)

### Phase 1 (빠른 체감 완성도)
- Guide 메인에 `EssentialsGrid` 추가
- `GuideCard`/섹션 컨테이너 규칙 적용
- 이미지 프레임/캡션 통일

### Phase 2 (경험이 “좋아지는” 동적 기능)
- QuickActionSheet(WiFi/체크인/BBQ/쓰레기)
- “이어보기” 저장/복원

### Phase 3 (난이도 상, 운영 가치)
- 개인화(예약 상태 기반 우선순위)
- 이벤트 로깅/지표 대시보드 기반 개선 사이클
