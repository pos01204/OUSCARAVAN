## 1) 스케일업 목표(디자인 중심)

### 성공 기준(기능 추가 없이 달성)
- **일관성**: 같은 의미는 같은 컴포넌트/스타일(예: 상태, 경고, CTA)
- **위계**: “제목→핵심값→보조설명→행동”이 1초 내 스캔됨
- **상태/피드백**: 로딩/빈 상태/성공/실패가 모두 같은 규칙으로 표현됨
- **절제된 동적 요소**: 고객에만 “짧은” 전환/피드백을 제한 적용(관리자는 최소)

---

## 2) 현재 프레임에서 “완성도를 깎는” 디자인 빈틈

### (A) 동일한 카드가 너무 많아 ‘중요도’가 평준화됨
- 카드가 모두 비슷하면, 고객은 “뭐가 중요한지” 느리게 판단합니다.
- 해결: **Card Variant**(기본/강조/경고/인터랙티브/정보)를 도입해 위계를 만든다.

### (B) 상태 표현(성공/주의/오류/진행)이 페이지마다 제각각
- 상태는 “텍스트+색+아이콘” 규칙이 고정되어야 합니다.
- 해결: **StatusBadge/StatusPill**을 단일화하고, 모든 페이지에서 재사용한다.

### (C) 인터랙션 피드백이 ‘있는 듯 없는 듯’하여 프리미엄 감이 약함
- 과하게 화려할 필요는 없지만, 눌림/호버/전환의 “마감”이 필요합니다.
- 해결: 고객은 **스프링/페이드**를 제한적으로, 관리자는 **미세한 hover**만 적용한다.

### (D) 로딩/빈 상태가 ‘임시 처리’처럼 보임
- Skeleton/Empty/Error는 UI 품질을 좌우합니다.
- 해결: **Skeleton 규격(높이/간격/시머)** + **EmptyState(문구+액션)**를 표준화한다.

---

## 3) 공통 디자인 토큰 & 컴포넌트 규격(가이드)

> 기존 라이트 테마/브랜드 팔레트는 유지합니다. (세부는 `UI_DESIGN_DIAGNOSIS_AND_IMPROVEMENTS.md` / `GUEST_VISUAL_DESIGN_PLAN.md` 참고)

### 3.1 컴포넌트 규격(권장)
- **Radius**: 카드/시트 `rounded-xl~2xl`, 버튼 `rounded-xl`, 칩/배지 `rounded-full`
- **Shadow**: 기본 `shadow-soft-sm`, 상위 `shadow-soft-md`, 강조(고객만) `shadow-soft-lg`
- **Border**: 기본 `border-brand-cream-dark/20~40`, 강조 `border-brand-cream-dark/60`

### 3.2 상태 컴포넌트 단일화
- **StatusBadge(관리자용)**: 정보 스캔용, 작고 선명하게
- **StatusPill(고객용)**: 감성 톤, 텍스트 친절하게(예: “곧 준비 시작해요”)

### 3.3 공통 “Empty / Error / Loading” 표준
- **EmptyState**
  - 제목: 1줄(현재 상황)
  - 설명: 1줄(다음 행동 유도)
  - 액션: 1개(재시도/탐색/문의)
- **ErrorState**
  - 심각도: info/warn/error 3단계
  - CTA: “다시 시도”는 기본 제공
- **Skeleton**
  - 카드 스켈레톤은 “실제 카드 레이아웃”을 닮게(행/열 수 맞춤)

## 4) 고객 페이지: ‘돋보이되 과하지 않게’ 컴포넌트 강화

> 프레임/섹션 구성은 유지하고, “카드/헤더/네비/피드백”의 마감을 올립니다.

### 4.1 Hero(환영): ‘한 번에 프리미엄 인상’
- **배경**: 크림 그라데이션 + 은은한 장식(blur blob 1~2개)
- **타이포 위계**: 인사(작게) → 이름(강조) → 보조 문구(짧게)
- **모션**: 페이지 첫 진입에만 1회(0.3~0.5s), 스크롤 중 반복 금지

### 4.2 QuickActionGrid: “아이콘+라벨” 품질 통일
- 아이콘만으로 의미 전달 금지(현재 준수) → 대신 **컬러칩/패딩/그림자**를 통일
- **터치 피드백**: `active:scale-[0.97]` 수준의 절제된 프레스만

### 4.3 InfoInspector(드로어/시트): 긴 정보도 정돈되게
- 헤더(제목/설명) + 본문(탭/페이지네이션) 구조 고정
- 스크롤 충돌 방지: 콘텐츠 영역만 스크롤, 헤더/탭은 고정

### 4.4 Toast/Success 피드백: “짧고 명확”
- 성공: 1줄(무엇이 되었는지) + 1줄(다음 안내)
- 실패: 1줄(무엇이 안 됐는지) + 1개 CTA(재시도/대체)

### 4.5 고객 카드 Variants(권장)
- **Primary CTA 카드**: 주문/체크인 같이 행동 유도
- **Info 카드**: WiFi/시간/안내처럼 정보 제공
- **Alert 카드**: 공지/주의(너무 강한 경고색은 지양)
- **Interactive 카드**: 목록(가이드/주문 내역) 클릭 가능

## 5) 관리자 페이지: ‘스캔 가능한 가독성’ 컴포넌트 강화

> 프레임(페이지 구조/기능)은 유지하고 “정보 밀도 조절 + 스캔성”만 올립니다.

### 5.1 페이지 헤더 표준(관리자)
- H1(명확한 제목) + 1줄 설명(작게) + 오른쪽 상단 1~2개 액션(새로고침 등)
- 부가 설명은 툴팁/HelpText로 분리(본문 혼잡 방지)

### 5.2 KPI 카드: 클릭 가능 여부가 즉시 보여야 함
- clickable 카드만 **미세한 hover 배경** + 키보드 포커스 링 강화
- 값(숫자)은 커지고, 힌트는 작아짐(현재 구조 유지 + 간격 정리)

### 5.3 리스트(카드) 정보 위계(현재 카드 리스트 유지 기준)
- 1줄: **예약자명 / 상태 배지**
- 2줄: 체크인·체크아웃 / 방 배정
- 3줄(필요 시): 연락처/메모
- 색은 최소 사용, 대신 굵기/정렬/간격으로 스캔성 확보

### 5.4 필터 UI: 복잡도를 ‘숨겨서’ 줄인다(레이아웃 유지 범위)
- 모바일: 필터를 시트로 “접기”
- 데스크톱: 인라인 유지(현재 구성 유지 가능)

## 7) 우선순위 로드맵(P0/P1/P2)

### P0 (가장 체감 큰 “마감” 작업)
- **Card Variants 규격화**(Info/CTA/Alert/Interactive)
- **StatusBadge/StatusPill 단일화**
- **Empty/Error/Skeleton 표준화**
- **고객 Hero 1회 진입 모션 + 프레스 피드백 통일**
- **관리자 헤더/CTA 배치 규격화**

### P1 (고객 ‘돋보임’ 강화, 과하지 않게)
- Guide/Order/Help 전체에 동일한 “클릭 가능 힌트” 패턴 적용
- InfoInspector(드로어/시트) 내부 탭/스크롤/패딩 마감 개선
- Toast 톤 정리(고객은 친절, 관리자는 간결)

### P2 (세련미 강화 옵션)
- 아이콘 세트 일관성 재점검(Lucide 유지 + 필요한 곳만 Iconify)
- 미세한 전환(페이지/탭) 애니메이션을 고객에만 제한 적용
