# 관리자 UI/UX 리팩토링 구현 작업 상세 (Phase 1: 예약/배정)

> **작성일**: 2026-01-19  
> **목표**: “월간 캘린더에서 직관적 파악(정보 밀도)” + “상단 컨트롤 과밀 제거(정보 밀림 방지)”  
> **범위**: 관리자 `/admin/reservations` 중심 (캘린더/리스트/필터)

---

## 1) 문제 정의(현장 피드백 기반)

### 1.1 캘린더 월간 뷰 정보 밀도 부족
- 월간 셀에 적재 가능한 예약 수가 적어 `+N 더보기`가 자주 발생
- `더보기 → 목록 모달`로 전환되며 “캘린더 컨텍스트”가 끊김
- 결과: 월간 예약 분포/미배정 위험을 한 화면에서 스캔하기 어렵다

### 1.2 예약/배정 화면 상단 컨트롤 과밀
- 뷰 토글/필터/UI가 상단에 누적되며 실제 정보(캘린더)가 아래로 밀림
- 프리셋 추가가 단순히 “컨트롤 더하기”가 되면 문제를 악화시킬 수 있음

---

## 2) 수용 기준(완료 조건)

### 2.1 월간 캘린더 KPI
- “더보기/목록 모달” 없이도 월간 화면에서 다음을 3초 내 스캔 가능:
  - 날짜별 **미배정 여부/건수**
  - 날짜별 **체크인/체크아웃 변동(건수)**
- 특정 날짜 상세 업무는 **인스펙터 1회**로 도달(모바일 드로어 / 데스크톱 사이드 시트)

### 2.2 컨트롤 과밀 방지
- 상단 컨트롤은 1단 고정:
  - `뷰 토글` + `프리셋 1줄(가로 스크롤)` + `필터 버튼`
- 검색/상태/날짜 범위/고급 옵션은 **필터 시트(2단)**로 이동

---

## 3) 구현 범위(변경 파일)

### 3.1 캘린더 개선(월간)
- `app/admin/reservations/ReservationCalendarView.tsx`
  - `showMore(+N)` 텍스트를 “요약 칩(미배정/체크인/체크아웃)” 중심으로 개선
  - `더보기` 클릭 시 **목록 모달** 대신 **인스펙터(드로어/시트)** 오픈
  - 날짜 헤더/셀에서 **요약 카운트**를 노출(월간 스캔 강화)

### 3.2 상단 컨트롤 정리
- `app/admin/reservations/ReservationFiltersClient.tsx`
  - 프리셋을 1줄(가로 스크롤)로 재구성
  - 필터는 시트로 이동(2차 위계)

---

## 4) 작업 체크리스트(누락 방지)

### 4.1 캘린더(월간)
- [ ] 셀에서 “미배정/체크인/체크아웃” 요약이 보이는가
- [ ] `+N 더보기` 클릭 시 캘린더 컨텍스트가 유지되는가(인스펙터)
- [ ] 모바일에서 인스펙터가 드로어로 열리며 스크롤/터치가 안정적인가
- [ ] 데스크톱에서 인스펙터가 사이드 시트로 열리며 캘린더가 유지되는가

### 4.2 상단 컨트롤
- [ ] 상단 컨트롤이 1단으로 유지되는가(높이 증가 금지)
- [ ] 프리셋은 1줄 가로 스크롤이고, 눌렀을 때 “업무 모드(리스트)”로 자연스럽게 전환되는가
- [ ] 필터 버튼에 “적용됨”이 표시되고, 시트에서 상세 조정이 가능한가

---

## 5) 리스크/완화
- **React Big Calendar 제약**: 셀 적재량을 무한히 늘릴 수 없음  
  → “상세 텍스트” 대신 “요약 칩 + 우선순위 노출”로 스캔 UX를 강화
- **모바일 화면 제약**: 월간 캘린더에 상세 텍스트를 많이 넣으면 가독성 저하  
  → 모바일은 “월간 파악 모드(요약 중심) + 업무 처리 모드(리스트 전환)”로 분리
