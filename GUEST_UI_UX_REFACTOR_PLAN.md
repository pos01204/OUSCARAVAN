# 고객(Guest) 페이지 UI/UX 진단 및 프론트엔드 리팩토링 기획안 (CX 최우선)

> **작성일**: 2026-01-19  
> **대상 경로**: `/guest/[token]/*` (홈/안내/주문·카페/도움말/체크인아웃)  
> **목표 우선순위**: 1) 고객 경험(CX)·감성 품질 2) 정보 위계·이해도 3) 전환(주문/가이드/문의) 4) 성능·접근성·인앱 안정성

---

## 1) 고객 UX 목표(프로덕트 관점)

### 1.1 고객의 “지금 필요한 것”을 3초 내 찾게 만들기
고객이 숙소에서 가장 자주 하는 행동은 다음 4가지로 수렴합니다.
- **즉시 사용**: WiFi 연결, 체크인/체크아웃, 위치/약도 확인
- **문제 해결**: 자주 묻는 질문/응급 연락/규칙 확인
- **즐길 거리**: 불멍/바베큐 가이드, 편의시설 이용
- **전환 행동**: 주문(배달/키오스크), 쿠폰/카페 이용

UI는 “페이지 나열”이 아니라 위 행동을 **정확한 순서와 위계**로 안내해야 합니다.

### 1.2 감성 품질 + 신뢰(안전) 동시 달성
- 고객 페이지는 운영 도구가 아니므로 **시각적 완성도(타이포/여백/이미지/모션)**가 만족도를 좌우합니다.
- 동시에 체크인/체크아웃, 주문, 응급 연락은 **신뢰·안전 영역**이므로 과한 장식보다 “명확함”이 우선입니다.

### 1.3 인앱 브라우저(카카오톡 등)에서도 “깨지지 않게”
이 프로젝트는 실제로 인앱 제약을 겪었습니다. 고객 페이지도 아래 원칙을 강제해야 합니다.
- Notification/ServiceWorker/Clipboard 등 미지원 기능은 **옵션**으로 취급(실패해도 UX가 망가지면 안 됨)
- 사용자에게 “대체 행동(복사 실패 시 수동 노출 등)”을 제공

## 3) 현재 UX 진단(강점/약점)

### 3.1 강점(좋은 기반)
- **정보의 풍부함**: 숙소 이용에 필요한 대부분의 정보가 앱 안에 존재
- **모바일 네비가 명확**: 하단 탭으로 “길 잃을 확률”이 낮음
- **동적 컴포넌트가 일부 잘 적용**:
  - `GuestOrderContent`의 탭 전환 애니메이션
  - `GuestHomeContent`의 hero motion
  - 주문/쿠폰/가이드 캐러셀 등 “체험 요소” 보유

### 3.2 개선 필요(고객 관점에서의 마찰)

#### (A) 홈에 정보가 너무 많아 “핵심 행동”이 묻힘
현재 홈은 카드가 많아, 고객이 “WiFi/체크인/약도”를 찾기 위해 스크롤을 많이 하게 될 수 있습니다.

#### (B) 동적 요소의 ‘일관된 규칙’이 부족
- 어떤 곳은 Dialog, 어떤 곳은 Drawer(vaul), 어떤 곳은 카드 확장 등 인터랙션 패턴이 혼재
- 결과: 앱 전체가 “하나의 제품”처럼 느껴지기보다 기능 조합처럼 느껴질 수 있음

#### (C) 인앱/저사양 환경에서의 리스크
예: `WifiCard`의 `window.innerWidth`/`innerHeight` 직접 참조는 SSR/특정 환경에서 리스크가 될 수 있고,
클립보드 실패 시 UX 대체가 제한적일 수 있습니다.

#### (D) “전환” 영역(주문/쿠폰)의 집중도가 더 올라갈 여지
주문은 매출/만족도에 직접 영향. 지금도 좋지만, CTA의 우선순위·피드백·재주문 흐름을 더 최적화할 수 있습니다.

## 4) 고객 UX 리팩토링 원칙(디자인/동작 규칙)

### 4.1 정보 위계(홈 기준)
- **1차(즉시 행동)**: WiFi / 체크인·체크아웃 / 약도(내 위치) / “관리자 연락”
- **2차(이용 정보)**: 안내/규칙/편의시설/FAQ
- **3차(부가)**: 예약상품 상세, 주문 내역 상세, 부가 위젯(일몰/카운트다운 등)

### 4.2 컴포넌트 규칙(일관성)
- **패턴 1: Quick Actions**(홈 상단 고정 영역)
  - 2~4개 큰 버튼(카드형)로 “WiFi/체크인/주문/도움” 즉시 이동
- **패턴 2: Cards with Primary CTA**
  - 카드 내부 CTA는 1개만 강하게(나머지는 보조)
- **패턴 3: Details Surface**
  - 모바일: Drawer(하단 시트)로 상세
  - 데스크톱: Dialog/Sheet로 상세
  - “같은 종류의 상세”는 같은 패턴으로 통일

### 4.3 모션/동적 요소 가이드(과하지 않게 고급스럽게)
- 모션은 **“행동 결과”와 “상태 변화”에만** 사용
  - 탭 전환, 성공 토스트, 카드 확장/접힘
- 콘페티/강한 이펙트는 “기쁨”이 명확할 때만(쿠폰 등) + 옵션/저감 모드 고려

### 4.4 인앱/접근성/성능 원칙
- Notification/Clipboard/Share 등은 **지원 여부 체크 + 대체 UX 제공**
- 이미지/캐러셀은 LCP에 영향을 주므로:
  - 첫 화면 이미지는 가볍게, 아래는 지연 로드
  - skeleton/placeholder를 일관되게 적용
- 터치 타겟 최소 44px 유지(버튼/탭/아코디언)

## 5) 화면별 리팩토링 기획(구체)

### 5.1 홈(`/guest/[token]`)
**문제**: 정보가 많아 핵심 행동이 분산될 수 있음  
**개선안**
- Hero 아래에 **Quick Action Row** 도입(카드형 2x2 또는 가로 스크롤):
  - `WiFi 연결`, `체크인/체크아웃`, `주문하기`, `도움말(응급 연락)`
- 홈의 카드 순서를 “행동” 중심으로 재정렬:
  1) Quick Actions
  2) 약도(배정된 공간)
  3) WiFi
  4) 체크인/체크아웃
  5) 주문 내역(최근 1~2개 요약 + 더보기)
  6) 나머지 위젯(일몰/시간 등)
- “예약 상품 및 옵션”은 기본 접힘(요약 금액/상품만 노출, 상세는 Drawer)

### 5.2 안내(`/guest/[token]/guide`)
**문제**: 카드 확장형이 길어질수록 스크롤 피로 증가  
**개선안**
- 카테고리는 1줄 가로 스크롤 칩으로 고정(높이 증가 방지)
- 가이드 상세는 카드 확장 대신 **Drawer/Sheet 인스펙터**로 통일
  - 목록에서는 제목/요약/주의/태그만
  - 상세(인스펙터)에서 이미지/FAQ/체크리스트/트러블슈팅
- BBQ 가이드는 “상단 고정 CTA”로 유지하되:
  - 캐러셀 진입 전 “소요 시간/준비물” 미니 요약 제공(기대치 관리)

### 5.3 주문·카페(`/guest/[token]/order`)
**문제**: CTA는 좋지만 ‘재주문/주문상태 확인’ 흐름을 더 강화 가능  
**개선안**
- 주문 탭 첫 화면을 “2개의 큰 카드(바베큐/키오스크)” 대신:
  - 상단에 **주문 상태 요약 바**(대기/준비/배송/완료)
  - 그 아래에 주문 CTA(2개)
- 주문 폼 모달은 단계형(선택 → 옵션 → 시간 → 확인)으로 UX 정교화(오류/로딩/성공 피드백 통일)
- 쿠폰 플립:
  - 콘페티는 1회성/옵션화(저감 모드)
  - “직원에게 보여주세요” 영역은 **스크린샷/밝기 최적화** 고려

### 5.4 도움말(`/guest/[token]/help`)
**문제**: 응급 연락처는 강하지만, FAQ 탐색성은 더 개선 여지  
**개선안**
- 상단에 “가장 많이 필요한 3개” 고정:
  - 관리자 전화 / 응급실 / WiFi 비번 잊음
- FAQ는 검색(키워드) + 카테고리(체크인/시설/주문/규칙)로 나눠 탐색성 강화
- 응급 연락처 버튼은 “지도/전화”의 시각적 우선순위를 통일(위험도에 따라 색/강조)

## 6) 컴포넌트/디자인 시스템 리팩토링(프론트엔드)

### 6.1 공통 컴포넌트 제안
- `GuestPageHeader`: 페이지 타이틀/설명/보조 CTA 통일
- `QuickActionGrid`: 홈 상단 즉시 행동 카드(아이콘/라벨/보조텍스트)
- `InfoInspector`: 모바일 Drawer / 데스크톱 Sheet로 “상세 보기” 통일
- `GuestStatusPill`: 주문 상태/체크인 상태/경고 배지 통일

### 6.2 스타일 토큰(규칙)
- 타이포: Heading(2단), Body(2단), Caption(1단)만 사용
- 카드: radius/spacing/shadow를 3단계로 제한(Primary/Default/Inset)
- CTA: primary 1개 원칙(한 화면에 primary 버튼은 최대 1~2개)

### 6.3 동적 컴포넌트 가이드
- Confetti/강한 애니메이션은 “쿠폰/축하”에만 제한 + 저감 모드
- Drawer/Dialog/Sheet 사용은 페이지마다 다르게 쓰지 말고 `InfoInspector`로 통일

## 7) 실행 계획(추천 순서)

### Phase G0 — UX 규칙/컴포넌트 기반 정리(0.5~1일)
- 공통 상세 표면(`InfoInspector`) 도입
- Quick Action 컴포넌트 도입(홈 우선)

### Phase G1 — 홈 재구성(1~3일)
- Quick Actions + 카드 순서 재배치
- 예약상품/옵션 정보 “요약/상세” 분리
- 주문 내역 “요약 + 더보기”로 정보 밀도 최적화

### Phase G2 — 안내/도움말 탐색성 개선(2~4일)
- 안내 상세를 인스펙터로 통일
- FAQ 검색/카테고리 강화

### Phase G3 — 주문 전환/재주문 최적화(2~4일)
- 주문 폼 단계형 UX 개선 + 상태 피드백 통일
- 쿠폰/카페 경험 시각 품질 업그레이드

## 8) 완료 기준(수용 조건)
- 고객이 앱 진입 후 3초 내에 **WiFi/체크인/주문/도움말** 중 하나에 도달 가능
- 인앱 브라우저에서 Clipboard/Notification 미지원이어도 **대체 UX로 완료 가능**
- 안내/FAQ에서 “찾기” 성공률(검색/카테고리)이 높아지고 스크롤 피로가 감소
