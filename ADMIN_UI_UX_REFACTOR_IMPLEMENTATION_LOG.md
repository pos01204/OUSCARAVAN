## 2026-01-19

### 구현 준비(요구사항 재확인)
- 캘린더 월간 뷰에서 “더보기/모달 의존”을 줄이고, 한 화면에서 **미배정/체크인/체크아웃**을 스캔 가능하게 개선
- 프리셋(업무 단축키)은 유용하지만 상단을 더 복잡하게 만들면 역효과 → **1단 고정 + 2단(시트) 분리** 원칙 적용

### 변경 예정 파일
- `app/admin/reservations/ReservationCalendarView.tsx`
- `app/admin/reservations/ReservationFiltersClient.tsx`

### 구현 내용(1차 반영)

#### 1) 캘린더 월간 정보 밀도 개선
- **날짜 헤더에 요약 칩 추가**: 미배정/체크인/체크아웃 건수를 날짜 셀 상단에서 바로 확인 가능
- **“더보기(+N)” 동작 변경**: 전체 화면 목록 모달 대신 “인스펙터”로 전환해 캘린더 컨텍스트 유지
- **인스펙터 UI**:
  - 모바일: `Drawer` 기반 하단 인스펙터
  - 데스크톱: `Sheet` 기반 우측 인스펙터

#### 2) 예약/배정 상단 컨트롤 과밀 해소
- 상단을 **1단(프리셋 1줄 + 필터 버튼)** 구조로 재정렬
- 상세 조건(상태/검색/체크인/체크아웃)은 **필터 시트(2단)**로 이동
- 필터 적용 시 `view=list`로 유도하여 “업무 처리” 흐름을 자연스럽게 유지

## 자체 평가/진단(업데이트 예정)

### 목표 대비
- **월간 스캔**: 날짜 헤더에서 미배정/체크인/체크아웃이 즉시 보이도록 개선되어 “무조건 더보기 클릭” 빈도가 감소
- **컨텍스트 유지**: “더보기/날짜 클릭”이 모달이 아닌 인스펙터로 열려 월간 캘린더 맥락이 유지됨
- **상단 과밀**: 프리셋을 1줄로 제한하고 상세 필터를 시트로 이동해 캘린더 본문이 덜 밀리도록 조정

### 좋아진 점(실무 관점)
- 월간 화면에서 “위험 날짜(미배정, 체크인/체크아웃 급증)”를 빠르게 스캔 가능
- 예약 상세 확인/배정 처리로 이동하는 경로가 “캘린더 유지” 상태에서 가능

### 남은 리스크 / 개선 여지
- React Big Calendar의 셀 높이 제약은 남아있어, “셀에 모든 예약명을 전부 노출”하는 방식은 구조적으로 불가능
  - 현재는 요약 칩으로 해결했지만, **셀 내 표시 우선순위(미배정 우선 노출)**를 더 강하게 적용하면 효율이 더 올라감
- 필터 시트가 `bottom` 고정이라 데스크톱에서는 우측 시트가 더 자연스러울 수 있음(사용성 테스트 후 조정 권장)

### 다음 단계 제안
- (Phase 1-후속) 캘린더 셀 내 이벤트 제목을 “업무형 축약 표기(미/입/퇴 + 이름)”로 표준화하여 추가 밀도 확보
- (Phase 1-후속) 인스펙터 내부에서 “미배정만 보기” 토글을 제공해 배정 업무를 더 빠르게
