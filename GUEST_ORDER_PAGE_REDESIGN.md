# 게스트 주문 페이지 재구성 기획 및 설계

## 1. 현황 분석

### 1.1 현재 문제점
- **"주문" 페이지의 목적 불명확**: 현재 페이지는 카페 메뉴 소개와 주문 기능이 혼재되어 있음
- **카페 이용 방식과 불일치**: 카페는 직접 방문하여 주문/수령하는 구조인데, 앱에서 카페 메뉴를 주문할 수 있는 것처럼 보임
- **실제 주문 가능 항목이 부각되지 않음**: 숯불/불멍 세트 및 키오스크 판매 물품 주문 기능이 메뉴 소개에 가려짐

### 1.2 현재 페이지 구성
```
주문 페이지 (/guest/[token]/order)
├── CouponFlip (10% 할인 카드)
├── MenuCarousel (카페 메뉴 소개 - 4개 카테고리)
├── 불멍/바베큐 주문 섹션
└── 카페 정보
```

### 1.3 사용자 요구사항
1. **주문 페이지의 목적**: 숯불/불멍 세트 및 키오스크 판매 물품에 대한 추가 주문
2. **카페 이용 방식**: 직접 방문하여 주문하고 수령 (앱 주문 불가)
3. **카페 메뉴**: 정보 제공용 (실제 주문은 카페에서 직접)

---

## 2. 재구성 방안

### 2.1 페이지 목적 재정의

#### 2.1.1 "주문" 페이지
- **주 목적**: 객실 배송 가능한 항목 주문
  - 불멍/바베큐 세트 주문 (기존 기능)
  - 키오스크 판매 물품 추가 주문 (신규)
- **부 목적**: 주문 내역 조회 및 상태 확인

#### 2.1.2 카페 메뉴 정보 처리
- **옵션 1**: "안내" 페이지에 카페 이용 안내 섹션 추가
- **옵션 2**: "주문" 페이지 하단에 "카페 이용 안내" 정보성 섹션으로 유지 (주문 불가 명시)
- **옵션 3**: 페이지 내부 탭으로 "주문"과 "카페 이용" 구분

**권장안**: 옵션 3 (스크롤 길이 문제 해결 및 사용자 편의성 균형)

## 3. 상세 설계

### 3.1 주문 페이지 재구성

#### 3.1.1 페이지 구조 (탭 방식)

**전체 구조**
```
주문/카페 이용 페이지 (/guest/[token]/order)
├── [상단] CouponFlip (10% 할인 카드) - 유지 (공통)
└── [탭 네비게이션]
    ├── [탭 1] 주문
    │   ├── 배송 주문
    │   │   ├── 불멍/바베큐 세트 주문
    │   │   └── 키오스크 판매 물품 주문 (신규)
    │   └── 주문 내역
    │       └── 주문 목록 및 상태 조회
    └── [탭 2] 카페 이용
        ├── 카페 메뉴 소개
        ├── 카페 위치 및 운영시간
        └── "카페에서 직접 주문 및 수령" 안내
```

**탭 방식의 장점**
- 스크롤 길이 문제 해결: 각 탭별로 콘텐츠 분리
- 명확한 정보 구분: 주문과 카페 정보를 시각적으로 분리
- 빠른 접근: 사용자가 원하는 정보에 즉시 접근 가능
- 탭 메뉴 이름 변경 가능: "주문/카페 이용"으로 명확히 표시

**대안 구조 (아코디언 방식)**
```
주문 페이지 (/guest/[token]/order)
├── [상단] CouponFlip (10% 할인 카드) - 유지
├── [섹션 1] 배송 주문
│   ├── 불멍/바베큐 세트 주문
│   └── 키오스크 판매 물품 주문 (신규)
├── [섹션 2] 주문 내역
│   └── 주문 목록 및 상태 조회
└── [섹션 3] 카페 이용 안내 (아코디언 - 기본 접힘)
    └── [클릭 시 확장]
        ├── 카페 메뉴 소개
        ├── 카페 위치 및 운영시간
        └── "카페에서 직접 주문 및 수령" 안내
```

**권장안**: 탭 방식 (스크롤 길이 문제 완전 해결)

#### 3.1.2 배송 주문 섹션

**불멍/바베큐 세트 주문** (기존 기능 유지)
- 세트 선택
- 수량 선택
- 배송 시간 지정
- 요청사항 입력
- 주문 제출

**키오스크 판매 물품 주문** (신규)
- **목적**: 객실 배송이 필요한 추가 물품 주문
- **카테고리 예시**:
  - 음료/간식
  - 생필품
  - 기타 용품
- **주문 프로세스**:
  1. 카테고리 선택
  2. 상품 선택 및 수량 지정
  3. 배송 시간 지정
  4. 요청사항 입력
  5. 주문 제출

#### 3.1.3 주문 내역 섹션

**기능**
- 최근 주문 목록 표시
- 주문 상태 확인 (대기 중, 준비 중, 배송 중, 완료)
- 주문 상세 정보 조회

**UI 구성**
- 주문 카드 형태로 표시
- 상태별 색상 구분
- 주문 번호, 주문 시간, 총액 표시
- 상세보기 버튼

#### 3.1.4 카페 이용 안내 탭 (탭 방식 적용 시)

**목적**: 카페 메뉴 정보 제공 (주문 불가 명시)

**구성**
- 카페 메뉴 소개 (기존 MenuCarousel 활용)
- **명확한 안내 문구**: "카페는 직접 방문하여 주문 및 수령해주세요"
- 카페 위치 및 운영시간
- 10% 할인 혜택 안내 (CouponFlip과 연계)
- 사업자 정보

**디자인**
- 탭으로 구분되어 별도 화면으로 표시
- 정보성 콘텐츠로 구성
- "정보" 아이콘 또는 배지 표시

#### 3.1.5 탭 네비게이션 구현

**UI 구성**
- 상단 고정 탭 바 (Sticky Tab Bar)
- 두 개의 탭: "주문" | "카페 이용"
- 활성 탭 하이라이트
- 부드러운 전환 애니메이션

**기술 구현**
- React Tabs 컴포넌트 활용 (shadcn/ui Tabs)
- URL 쿼리 파라미터로 탭 상태 관리 (선택사항)
- 모바일 최적화된 탭 디자인

## 4. UI/UX 개선사항

### 4.1 탭 네비게이션 디자인

**탭 바 스타일**
- 상단 고정 (Sticky) 또는 페이지 상단 배치
- 모바일: 전체 너비, 큰 터치 영역
- 데스크톱: 중앙 정렬, 호버 효과
- 활성 탭: Primary 색상, 하단 보더 또는 배경색
- 비활성 탭: Muted 색상

**탭 전환 애니메이션**
- 부드러운 슬라이드 또는 페이드 전환
- 전환 시간: 0.3초
- Framer Motion 또는 CSS Transition 활용

### 4.2 콘텐츠 최적화

**주문 탭**
- 배송 주문 섹션: 주요 액션 영역, 강조된 디자인
- 주문 내역 섹션: 정보 표시 영역, 카드 형태
- 스크롤 최적화: 각 섹션별 적절한 간격

**카페 이용 탭**
- 정보성 콘텐츠로 구성
- "직접 방문 주문" 안내 문구 강조
- 메뉴 소개는 캐러셀 또는 그리드 형태
- 카페 정보는 카드 형태로 정리

### 4.3 스크롤 길이 최적화

**문제 해결 방안**
1. **탭 방식 (권장)**: 콘텐츠를 두 개의 탭으로 분리하여 각 탭의 스크롤 길이 최소화
2. **지연 로딩**: 카페 메뉴 이미지 지연 로딩 (Lazy Loading)
3. **가상화**: 주문 내역이 많을 경우 가상 스크롤 적용 (선택사항)
4. **아코디언**: 카페 안내 섹션을 접을 수 있게 구성 (대안)

### 4.4 사용자 안내
- 각 탭에 명확한 제목과 설명 추가
- 카페 이용 탭에 "직접 방문 주문" 안내 문구 강조
- 주문 가능/불가능 항목 시각적 구분

### 4.5 접근성
- 탭별 ARIA 레이블 명확화
- 키보드 네비게이션 지원 (Tab, Arrow Keys)
- 스크린 리더를 위한 설명 추가
- 탭 전환 시 포커스 관리

## 11. 스크롤 길이 문제 해결 방안 상세

### 11.1 문제 분석
- 카페 메뉴 소개 (4개 카테고리) + 주문 섹션 + 주문 내역 + 카페 정보
- 예상 스크롤 길이: 모바일 기준 약 3000-4000px (화면 높이의 5-7배)
- 사용자 피로도 증가 및 정보 탐색 어려움

### 11.2 해결 방안 비교

#### 방안 1: 탭 방식 (권장) ⭐
**구조**
- 페이지 내부에 탭 네비게이션 추가
- "주문" 탭과 "카페 이용" 탭으로 분리
- CouponFlip은 상단 공통 영역

**장점**
- ✅ 스크롤 길이 문제 완전 해결 (각 탭별 독립적 스크롤)
- ✅ 정보 구분이 명확함
- ✅ 사용자가 원하는 정보에 빠르게 접근
- ✅ 탭 메뉴 이름 변경 가능 ("주문/카페 이용")

**단점**
- 탭 전환 필요 (하지만 이는 장점이기도 함)
- 구현 복잡도 약간 증가

**예상 스크롤 길이**
- 주문 탭: 약 1500-2000px
- 카페 이용 탭: 약 1500-2000px
- 각 탭이 독립적으로 관리되어 사용자 경험 향상

#### 방안 2: 아코디언 방식
**구조**
- 카페 안내 섹션을 기본 접힘 상태로 배치
- 클릭 시 확장

**장점**
- 구현이 간단함
- 한 페이지에서 모든 정보 확인 가능

**단점**
- 여전히 스크롤 길이가 길 수 있음
- 탭 메뉴 이름 변경만으로는 부족

**예상 스크롤 길이**
- 접힘 상태: 약 2000px
- 확장 상태: 약 3500px

#### 방안 3: 별도 페이지 분리
**구조**
- 카페 이용을 별도 페이지로 분리
- `/guest/[token]/cafe` 경로 추가

**장점**
- 완전한 분리
- 각 페이지의 목적이 매우 명확

**단점**
- 네비게이션 복잡도 증가
- 탭 메뉴 구조와 맞지 않음

### 11.3 최종 권장안: 탭 방식

**이유**
1. 스크롤 길이 문제를 가장 효과적으로 해결
2. 탭 메뉴 이름 변경 요구사항과 일치
3. 사용자 경험과 운영 효율성 균형
4. 모바일 앱에서 일반적인 패턴

**구현 우선순위**
1. 탭 구조 구현 (Phase 1)
2. 각 탭 콘텐츠 구성
3. 탭 전환 애니메이션
4. 모바일 최적화
